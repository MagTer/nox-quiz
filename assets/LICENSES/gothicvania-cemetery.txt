Processing note: the far (horizon-glow sky) and mid (mountains) parallax
layers were retinted via `hue_shift_band()` (copied verbatim from
.planning/research/v6-scouting/styleboard.py, same call already proven in
styleboard.py's cemetery()) — `hue_shift_band(195, 245, -50)` — because the
pack's native horizon glow is magenta, which violates this project's no-pink
rule as-is. Pre-retint measurement: the raw sky background.png scored ~78.7%
dominant pink/magenta hue. Post-bake (retint applied, then run through the
shared `_remap_luminance` palette-conform pass), the far layer measures
0.0000 (0%) via scripts/lib/pink_scan.py.

The terrain atlas was NOT retinted — direct pixel sampling of both candidate
crop regions found the full tombstone sheet's ~8.7% pre-remap pink-hue figure
was entirely attributable to a single dark violet/plum tombstone SHADOW tone
(RGB (65,25,59)), the same low-brightness HSV hue-instability class already
documented for assets/player-swamphunter.png's pink-scan allowlist entry —
not a genuinely pink surface. The final crop rects (grass-tuft/rock region,
0.9%/0.0% pre-remap) avoid this artifact entirely and require no retint.

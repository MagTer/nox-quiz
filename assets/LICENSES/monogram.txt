File provenance note (read before assuming a filename mismatch is an error):
The itch.io page's own Uploads list names 4 files for this product —
monogram.zip, monogram.ttf (10 kB), monogram.p8, and monogram-extended.ttf
(57 kB). All 4 ship under the one CC0 declaration above; "-extended" is an
official companion file from the SAME product/page (a superset glyph
coverage variant), not a different or third-party font. itch.io's plain
`monogram.ttf` sits behind an interactive, session-gated "Download Now"
purchase-flow (pay-what-you-want, min price 0) that a stable, scriptable URL
does not front. The page's OWN live custom CSS, however, embeds a direct,
non-interactive `@font-face` URL to `monogram-extended.ttf`
(https://menez.io/css/font/monogram-extended.ttf — menez.io is the same
author's asset-hosting domain, used by the page itself to render its own
description text in the font) — this is the "embedded direct file URL"
vendored here, per this plan's sourcing fallback order. The downloaded file
was verified before vendoring: TrueType-valid (openable via
`PIL.ImageFont.truetype`), and its internal font-name table reports
`('monogram', 'Regular')` — confirming it is genuinely the Monogram font,
not a mislabeled substitute. Saved locally as `monogram.ttf` (the generic,
code-facing name every other vendored asset in this repo uses, matching
`scripts/build-art-assets.py`'s `FONT_PATH` reference) rather than
`monogram-extended.ttf`, to avoid implying a second, separate font asset
exists in this project.

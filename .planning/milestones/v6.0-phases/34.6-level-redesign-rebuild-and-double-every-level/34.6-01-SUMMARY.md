## Decisions Made
- `clearLevel({ table } = {})` distinguishes the two paths on `table !== undefined` rather than a separate flag — the math path always supplies `{ table }` (from the answered question's `q.a`), the key path always supplies `{}`. This keeps the function signature identical to the pre-extraction `onClear({ table })` contract while adding the key branch with zero new parameters.
- `XP_KEY_SKIP` defaults to 20 (flat "full value," matching `XP_HARD`) per 34.6-RESEARCH.md's A1 recommendation — the alternative (drawing a representative `allowedTables` band) was not chosen because a flat constant is simpler and equally defensible as "no penalty." This value is explicitly re-confirmable at the levels-01/02 human-verify checkpoint in plan 34.6-04, not locked yet.
- The key-path banner/sfx are inlined in `game.js` rather than exposed as a new `mathGate.js` export, because `heldKeyIds` is scene-closure run-state that `mathGate.js` has no reference to — widening that module's API for a two-line render would be a larger seam change than duplicating four `add([...])`/`audio.playSfx` lines (RESEARCH's explicit reasoning for keeping the branch in `game.js`).

<objective>
Land the ONE code change of Phase 34.6: make the end-of-level gate key-conditional. On levels that place a math-skip key, a held key clears the level with full XP INSTEAD of opening the end math challenge; with no key held the math challenge runs exactly as today. Extract the level-clear body into a single shared `clearLevel()` closure so the math path and the key path cannot drift on XP, unlock, or persistence.

Purpose: This is the mechanic seam every even (2/4/6/8) rebuilt level depends on — the key becomes an optional reward on a risk/reward high route (LEN-02). It reuses Phase 34.5's shipped key pickup + `heldKeyIds` run-state verbatim; only the goal branch and one extracted function are new.

Output (artifacts this plan produces):
- `clearLevel({ table })` closure function in `src/scenes/game.js` (shared by both clear paths).
- The key-held branch inside `onReachGoal()`.
- `CONFIG.PROGRESS.XP_KEY_SKIP` config key (the full-XP-on-key-skip value; A1 default = 20, confirmable at the 01/02 checkpoint in plan 34.6-04).

Note — no level geometry changes here. This plan is code + config only. It leaves the CURRENT throwaway level-02 key+lock in place (rebuilt later in 34.6-03); a driven player who collects that key and reaches the goal will now free-clear it, which is harmless (no gate asserts otherwise, and that level is discarded next wave).
</objective>

    For the 3 standard biomes (`levelData.biome !== "cemetery"`, i.e. swamp/town/castle —
    Castle renders through this exact same path, no special-casing per 32-CONTEXT.md's
    explicit "render as-is" instruction): first chunk-emit the fill body — for `cx` from
    `runX` to `runX + runW` in steps of `FILL_CHUNK_COLS * T`, add one
    `sprite(atlasSprite, { frame: FILL_FRAME, tiled: true, width:
    Math.min(FILL_CHUNK_COLS * T, runX + runW - cx), height: fillDepthPx })` at `pos(cx,
    runY + T)`, tagged `"ground-fill"`. Then emit the cap row — for `tx` from `runX` to
    `runX + runW` in steps of `T`, add one `sprite(atlasSprite, { frame: CAP_FRAME })` at

    starts one tile-row EARLIER — `pos(cx, runY)` instead of `pos(cx, runY + T)` — with

    despite starting a row higher), emitted BEFORE the cap-row loop so the cap tiles draw
    on top as a decorative overlay (per `docs/LEVEL-DESIGN.md` §9's explicit follow-up:
    Cemetery's cap frame is transparent in rows 0-9, so without an always-solid fill
    underneath the ENTIRE cap-row cell — not just the row below it — that transparency

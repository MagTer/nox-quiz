<objective>
Remove the collect-the-answer mechanic everywhere, atomically, per MECH-01: the mechanic's
code, its config block, its level data on the 5 levels that carried it, and every harness
fixture that defended or exercised it — in one coherent change, leaving the tree in a fully
green state at every step (no intermediate commit where one side references a thing the
other side deleted). Also verifies (MECH-02, documentation-only — no code change) that the
5 affected levels still deliver a satisfying math rhythm using the remaining mechanics.

Purpose: the collect-the-answer mechanic never landed well (see PROJECT.md / v5.0 backlog
999.1) and per SEED-001/v6.0 scope, no replacement 4th mechanic is being built — the game
is better with three well-tuned mechanics (door/checkpoint-gate/enemy) than four, one of
which was always weak. This plan is the "cut it" half of that decision, executed so nothing
is left half-deleted.
Output: collect.js gone; CONFIG.COLLECT gone; 5 level descriptors' collect data gone; the
build.js collect-zone builder gone; game.js's wireCollect wiring gone; every harness
fixture (check-gate.sh assertion #13, 3 smoke-progress golden fixtures, reachability.mjs/
over-hole-check.mjs/mechanic-drive.mjs kind-lists, check-import-safety.sh file lists)
updated to match. Full gate suite green.
</objective>

**KEY HALF — analog `src/mechanics/secretAlcove.js:41-89`.** Copy the fire-once `Set` latch
and the feedback triad; DROP all persistence/replay-guard (key-held is run-state, never
serialized). Imports pattern (secretAlcove.js:19-21):
```javascript
import { CONFIG } from "../config.js";
import * as fx from "../fx.js";

```
Fire-once latch + feedback (secretAlcove.js:47-88, stripped of `progress.*` / `hasSecretFound`):
```javascript
const collected = new Set();                 // closure-local (never module-level) — GC'd with scene
player.onCollide("key", (keyObj) => {
  if (collected.has(keyObj)) return;
  collected.add(keyObj);
  onPickup();                                 // flips scene keyHeld = true
  hud.showKey();                              // persistent HUD indicator
  fx.pop(keyObj.pos.clone());                 // clone .pos BEFORE destroy (game.js coin idiom)
  fx.popupText(keyObj.pos.clone(), "+ KEY");  // was "+5 XP" in the alcove
  audio.playSfx("pickup");                    // reuse existing sfx — no new asset
  destroy(keyObj);                            // destroy LAST
});
```

**LOCK HALF — analog `src/mechanics/door.js:38-86`.** Copy the `opened` Set + the
destroy-before-continue ordering. DROP the whole freeze machinery: `door.js` has
`let busy` (54), `player.vel = vec2(0)` + `player.paused = true` (61-62), `openChallenge({...})`
(64), and `player.paused = false` (82) — the lock has NONE of these (it never freezes; it's a
wall). Replace `openChallenge` with a `hasKey()` branch. Keep the load-bearing cleanup ordering
from door.js:74-79:
```javascript
const opened = new Set();
let hintShowing = false;                       // no-timer fire-once gate for the hint
player.onCollide("lock", (lockObj) => {
  if (opened.has(lockObj)) return;
  if (hasKey()) {
    opened.add(lockObj);
    audio.playSfx("door");                     // reuse door sfx
    fx.clearBurst();
    destroy(lockObj);                          // collider FIRST (door.js:77 ordering)
    if (lockObj.panelObj) destroy(lockObj.panelObj);   // door.js:78 stashed-visual cleanup
    return;
  }
  // No key: solid wall stays; brief non-blocking hint (NO pause, NO challenge).
  if (hintShowing) return;
  hintShowing = true;
  const hint = add([
    text(CONFIG.LOCK.HINT_TEXT, { size: CONFIG.LOCK.HINT_SIZE }),
    anchor("center"), pos(center().x, CONFIG.LOCK.HINT_Y),
    color(...CONFIG.PALETTE.TEXT), fixed(), z(9200), opacity(1),
    "hud-flash",                               // torn down on scene leave (anti-leak)
  ]);
  tween(1, 0, CONFIG.LOCK.HINT_MS / 1000, (v) => (hint.opacity = v), easings.easeOutQuad)
    .onEnd(() => { destroy(hint); hintShowing = false; });   // no-timer self-clean
});
```
`wireKey` signature (matches game.js injection): `wireKey({ player, hud, onPickup, hasKey })`.

## Shared Patterns

### a727c13 engine-global discipline
**Source:** `src/mechanics/secretAlcove.js:11-13`, `src/mechanics/door.js:12-14` headers.
**Apply to:** `src/mechanics/key.js` (all engine globals INSIDE `wireKey()` body) AND
`scripts/lib/key-lock-check.mjs` (pure/node-importable — zero engine globals). Enforced by the
gate whose file list you also edit.

### Closure-local fire-once + anti-leak
**Source:** `secretAlcove.js:47` (`const found = new Set()`), `door.js:41` (`const opened`),
`game.js:59/63` (`let coinsCollected`/`goalReached`).
**Apply to:** `key.js` (`collected`/`opened` Sets, `hintShowing` bool) and `game.js` (`keyHeld`).
Never module-level; GC'd with the scene; run-state never serialized.

### No-timer self-clean
**Source:** `door.js` `tween().onEnd()` cleanup; `hud.js:90` explicit "no scheduler" note.
**Apply to:** the lock's hint fade — `tween(...).onEnd(() => { destroy(hint); hintShowing = false; })`.
NEVER `setTimeout`/`wait()`/`loop()` (banned by `check-safety.sh`).

### `?? []` never-brick guard
**Source:** `build.js:266/364`, `over-hole-check.mjs:49/58`.
**Apply to:** every new `keys`/`locks` iteration in build.js AND key-lock-check.mjs, so all 8
existing levels (no keys/locks yet) still build and validate unchanged.

### Two-tier HARD-FAIL/WARN + failures counter
**Source:** `over-hole-check.mjs` result contract + `validate-levels.mjs:99-103`.
**Apply to:** `key-lock-check.mjs` rows and its composition — `hardFailCount` sums into
`failures`, exit(1) if any.

### Reuse existing SFX / colored-rect placeholders (no new assets)
**Source:** alcove uses `pickup`, door uses `door` sfx; art deferred to Phase 35.
**Apply to:** key pickup (`pickup`), lock open (`door`), and key/lock-panel visuals
(colored rects) — keeps `check-assets-manifest.mjs` and `main.js` loads untouched.

## Corrections to RESEARCH assumptions (verified this session)

1. **`bestMarginToPoint` is ALSO module-private** in `reachability.mjs` (line 406), not just
   `bfsWithPathMargin` (line 323, as RESEARCH A1/Wave-0 noted). BOTH need a `export` added, or
   the `import { ... bestMarginToPoint ... }` in key-lock-check.mjs will be `undefined`. Trivial
   one-word change per line; re-run reachability.mjs's self-test after.
2. **`build.js` lock panel should be a colored rect, not `sprite("door")`.** The door panel
   analog (build.js:285) uses `sprite("door")`; the lock must NOT reuse that sprite (wrong art)
   and must NOT declare a new sprite (manifest churn — Pitfall 4). Use `rect(...)` +
   `color(...CONFIG.LOCK.PANEL_COLOR)` for the placeholder.

<objective>
Make the static coin model falsifiable in the real engine. Plan 34-01's model is a static graph — and a static model is precisely what failed to catch this bug class for four milestones. This plan builds `scripts/audit-coins.mjs`: for every coin the model calls reachable, it places the real player on the witness's launch surface, drives the real keyboard input surface, and requires the real coin `area()` to fire the real `onCollide("coin")` handler in `src/scenes/game.js`.

The model is deliberately obstruction-blind (it does not know about platform undersides, spike hits, or blockers). That is a licence to over-credit. This gate is the thing that catches it: a coin the model PASSes but the engine refuses to collect is a RED that must be fixed in the model or fixed by moving the coin.

Purpose: no coin is ever "verified" by a graph alone. Checks that don't play the game lie.
Output: a permanent, manually-invoked in-engine coin gate, run green against the coins the model already passes on all 8 pre-move levels.
</objective>

    3. For each coin WITH a witness, replay it:
       - If the coin is already gone from `get("coin")` at attempt start (an earlier coin's drive swept through it), record `collected: true, via: "incidental"` — it was still genuinely collected in-engine by a real driven player, which is exactly the claim under test.
       - Otherwise, teleport the player to the witness's launch surface using `repositionAndSettle`'s idiom: `p.pos.x = takeoffX + offset; p.pos.y = witness.launchY - 32; p.vel = vec2(0);` then settle 300ms.
       - Drive by family: `walk` -> hold the direction key (`ArrowRight` for dir +1, `ArrowLeft` for dir -1) ~900ms. `fall` -> hold the direction key ~1300ms (the player walks off the pinned edge and falls). `jump` -> hold the direction key AND hold Space for the whole flight, ~1300ms (hold, never tap — releasing while rising triggers `CONFIG.JUMP_CUT` and truncates the arc; `calibrate-jump-envelope.mjs` documents exactly this).
       - Sample every ~16ms; succeed as soon as that specific coin (matched by its `{x, y}` against `get("coin")`) is gone. Always release keys in a `finally`.
       - Takeoff sweep: the static model budgets horizontal travel at the shaved envelope speed (218 px/s) while the engine actually runs at `CONFIG.RUN_SPEED` (240 px/s), so the true pass-through point sits slightly further along the arc. Retry the same witness across offsets `[0, +12, -12, +24, -24]` (clamped to stay on the launch node), succeeding on the first offset that collects the coin. This sweep absorbs the speed difference; it does not invent a new reachability claim — every attempt still starts on the witness's own spawn-reachable launch surface and uses only real key input.

    4. Print a JSON results array (level, coin index, coin {x,y}, model status, witness family/dir, collected, via, offsetUsed) and a final summary line.

    Exit code contract (this IS a pass/fail gate, unlike `audit-phase21-mechanics.mjs`): exit 1 if ANY coin with a model witness was NOT collected in-engine; print `AUDIT-COINS: FAIL` and list the offenders. Otherwise print `AUDIT-COINS: ALL WITNESSED COINS COLLECTED` and exit 0. Also always print the count of model-HARD-FAIL coins that were skipped, so the run's output is a complete picture of the level set.

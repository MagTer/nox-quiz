# Phase 34.6.1 Plan 03: Sprite Wiring + Key Framing Summary

**Wired the vendored coin/goal/key sprites into build.js with footprint-pinned colliders (goal 16px, key 20px via `sprite({width,height})` + bare `area()`, never `scale()`), and reworded the key pickup popup + HUD indicator to plain-ASCII "SKIP KEY" framing sourced from config.**

## Accomplishments
- `main.js` registers a new static `loadSprite("key", "../assets/key.png")` alongside the unchanged coin/goal loads (34.6.1-02 already overwrote those PNGs in place at existing geometry).
- `config.js` gained `CONFIG.KEY.PICKUP_LABEL` ("+ SKIP KEY") and a reworded `CONFIG.HUD.KEY_GLYPH` ("SKIP KEY", was bare "KEY"); `CONFIG.KEY.COLOR` documented vestigial; `KEY.W/H` (20) and `COIN_FRAMES` (8) untouched.
- `build.js`'s goal builder now renders `sprite("goal", { width: CONFIG.GOAL_SIZE, height: CONFIG.GOAL_SIZE })` and the key loop renders `sprite("key", { width: CONFIG.KEY.W, height: CONFIG.KEY.H })` (replacing the old `rect(CONFIG.KEY.W, CONFIG.KEY.H)`) — both paired with a bare `area()` so the 16px/20px colliders stay pinned to the rendered size, independent of the source PNGs' native dimensions. The `?debug=1` magenta key marker still applies its tint over the sprite; the non-debug branch drops the now-superseded gold `CONFIG.KEY.COLOR` fill.
- `key.js`'s pickup popup now sources `CONFIG.KEY.PICKUP_LABEL` instead of the hardcoded `"+ KEY"` string — a single-line change; the rest of the `onCollide("key")` handler (onPickup/hud.showKey/fx.pop/audio.playSfx/destroy ordering) is byte-unchanged, and the key->skip mechanic (`heldKeyIds`/`endSkipArmed`/`XP_KEY_SKIP` in `game.js`) was not touched.

<objective>
Build the secret-alcove XP-bonus mechanism end-to-end (LVL-06's mechanics layer): a new `progress.js` method that can award a flat, non-table-scaled XP amount, a new `src/mechanics/secretAlcove.js` collision handler (fire-once, no challenge, no freeze -- unlike every other mechanic), and the `build.js`/`game.js` wiring that turns a level's `geometry.secretAlcove` array into a real, silent, walk-through bonus in the running game.

This plan builds the MECHANISM ONLY -- no level descriptor references `secretAlcove` yet. Levels 5-8 (Plan 25-03) and the retrofit into levels 1-4 (Plan 25-04) both depend on this landing first, per 25-RESEARCH.md's explicit build-order recommendation ("(1) before (2)/retrofit").

Purpose: 25-CONTEXT.md's "Secret XP Alcove" decision requires every level (1-8) to reward a flat 5 XP bonus on touch, deliberately below XP_EASY (10), with zero punishment for missing it. 25-RESEARCH.md's Pattern 2(b) finding is load-bearing: this literally cannot be expressed by the existing `progress.addXp(table)` signature, because `calculateXp(table)` can only ever return `CONFIG.PROGRESS.XP_EASY` (10) or `XP_HARD` (20) -- there is no `table` value that yields 5. A genuinely new sibling method is required, not a call-site reuse or a fabricated table hack.
Output: `src/progress.js` gains `addBonusXp(amount)` (same closure, same carry-over while-loop as `addXp`). `src/config.js` gains `CONFIG.PROGRESS.XP_ALCOVE: 5`. New `src/mechanics/secretAlcove.js` exports `wireSecretAlcove({ player, progress })`. `src/levels/build.js` gains a `secretAlcove` geometry-array loop with NO blocker collider (a walk-through trigger, unlike doors/gates/enemies, which all need an apex-derived tall anti-jump-over blocker). `src/scenes/game.js` wires it with `progress` (not `brain`) -- the only mechanic call site with this parameter shape. `scripts/check-import-safety.sh` adds the new module to its a727c13 negative-scan scope, matching how door.js/gates.js/enemy.js/collect.js are already scoped there.
</objective>

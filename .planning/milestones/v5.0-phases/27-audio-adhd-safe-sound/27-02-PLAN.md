  <action>
Create src/audio.js as a new module mirroring src/progress.js's file-header discipline: document (1) the a727c13 rule — every Kaplay primitive (loadSound, loadMusic, play, setVolume, getVolume, onKeyPress, add, text, color, fixed, z) is referenced ONLY inside function bodies, never at module top level; (2) that musicHandle and muted are the ONE sanctioned exception to this codebase's closure-local-state convention (mirroring game.js's onHide precedent) because music playback must survive go() scene transitions; (3) the guarded storage seam mirrors progress.js's storageAvailable()/try-catch/never-throw discipline exactly, using its OWN localStorage key (CONFIG.AUDIO.MUTE_STORAGE_KEY) and never touching CONFIG.SAVE.KEY. Import only CONFIG from "./config.js".

Declare two module-level `let` bindings: musicHandle (null initially — plain data reference, not an engine call, so safe at module scope) and muted (null initially — lazily loaded on first read).

Implement the guarded storage helpers exactly like progress.js's shape: storageAvailable() (try/catch around a typeof localStorage check, returns boolean, never throws), loadMuteFlag() (returns false if storage unavailable; wrapped in try/catch; reads CONFIG.AUDIO.MUTE_STORAGE_KEY and returns true only on a strict === "1" comparison — never JSON.parse a mute flag, it is a plain string, not a blob), writeMuteFlag(v) (no-op if storage unavailable; try/catch around localStorage.setItem, catch block warns via console.warn and never rethrows, mirroring writeSave()'s forgiving mandate).

Implement isMuted(): if `muted` is still null, lazily set it via loadMuteFlag(), then return it.

Implement ensureMusicPlaying(): if musicHandle is already truthy, return immediately (the idempotency guard that is the entire point of this pattern — it is the ONLY thing standing between this codebase and the exact music-stacking bug AUD-04 exists to prevent). Otherwise call play("ambient", { loop: true }) and store the returned handle in musicHandle, then set musicHandle.volume = CONFIG.AUDIO.MUSIC_VOLUME as a property assignment AFTER creation (the music-path internal helper only reads `loop`/`paused` from the options object passed to play() — a `volume` key inside that object is silently ignored per 27-RESEARCH.md Pitfall 1, so it must be set on the returned handle instead).

Implement playSfx(name, vol): calls play(name, { volume: vol ?? CONFIG.AUDIO.SFX_VOLUME }). Keep vol in the conventional 0..1 range in every call site (the SFX-path gain setter has no upper clamp, unlike the music path).

Implement an internal applyMuteState() helper that calls setVolume(isMuted() ? 0 : 1) — this is NOT in 27-PATTERNS.md's snippet but is REQUIRED for AUD-03's literal "the setting persists across reloads" to actually be true: without it, a page reload with a previously-persisted muted=true flag would show the mute icon as muted but leave the master gain at its default (unmuted) value, since only toggleMute() calls setVolume() in the pattern reference. Call applyMuteState() as the first statement inside wireAudioUI() below, so every scene mount (including the very first one on a fresh page load) re-syncs actual audio output to the persisted flag. This is idempotent and cheap to call on every scene mount.

Implement toggleMute(): flips `muted` (lazily loading it first via isMuted() if still null), calls setVolume(muted ? 0 : 1) (the ONE master-gain call that silences/restores music AND every SFX instance simultaneously — see 27-RESEARCH.md Pattern 3), then writeMuteFlag(muted).

Implement wireAudioUI() — the per-scene mount function every scene body (title.js, select.js, game.js) calls fresh: first call applyMuteState() (see above), then register onKeyPress(CONFIG.AUDIO.MUTE_KEY, () => { toggleMute(); refresh the icon text }), then add() a small fixed()+high-z() text() icon at CONFIG.AUDIO.ICON_X/ICON_Y showing "MUTE" when isMuted() is true or "SND" otherwise, colored via CONFIG.PALETTE.TEXT_DIM, sized via CONFIG.AUDIO.ICON_SIZE, tagged so it tears down with its own scene (mirror hud.js's "hud" tag idiom, or the scene's own tag if called from title.js/select.js — this function does not need a "hud" tag specifically since it must work in scenes that never mount hud.js). Keep a reference to the added icon object so the onKeyPress handler can update its .text property after toggling — do NOT create a new icon object on every mute-toggle press (a fresh add() per press would leak).

Export ensureMusicPlaying, toggleMute, isMuted, playSfx, wireAudioUI.
  </action>

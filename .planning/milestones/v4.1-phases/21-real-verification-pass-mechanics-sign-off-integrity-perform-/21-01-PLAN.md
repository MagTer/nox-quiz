    `newContext({ viewport: { width: 960, height: 540 } })` skeleton verbatim. Set `PORT` to
    `8768` (8765/6/7 are already taken by browser-boot.mjs/screenshot-phase18.mjs/

    At the top of the script (Node-side, outside the browser page context — this module is pure
    data and safe to import directly), add `import { getLevel, LEVEL_ORDER } from "../src/levels/index.js";`.

    Define `deriveGapRanges(geometry)`: sort `geometry.floors` by `x` ascending; for each adjacent
    pair, if `next.x > prev.x + prev.w`, emit `{ start: prev.x + prev.w, end: next.x }`; return the
    array of gap ranges (empty if the level has no gaps).

    Define `deriveEncounters(geometry)`: build one array merging `geometry.doors` (tag `"door"`,
    `renderChoices: true`), `geometry.mathGates` (tag `"math-gate"`, `renderChoices: true`),
    `geometry.enemies` (tag `"enemy"`, `renderChoices: true`), and `geometry.collectZones` (tag
    `"answer-zone"`, `renderChoices: false`) — each entry keeps `{ x, tag, renderChoices }|`
    (guard every array with `?? []` since some levels omit some mechanics); sort ascending by `x`;
    return it.

    Define `async function driveToX(page, targetX, gapRanges)`: `page.keyboard.down("ArrowRight")`;
    loop up to 80 iterations, each iteration `await page.waitForTimeout(100)` then
    `const x = await page.evaluate(() => { const p = get("player")[0]; return p ? p.pos.x : null; })`;
    if `x` falls within 24px before the `start` of any gap range (and past that range's own
    already-jumped state — track a simple `jumped` Set of gap starts to avoid repeat taps),
    `await page.keyboard.press("space")`; break the loop early (before releasing ArrowRight) if
    `x !== null && x >= targetX - 16`, or if `await page.evaluate(() => get("challenge").length > 0)`
    is already true; after the loop (whether by break or the 80-iteration cap), always
    `await page.keyboard.up("ArrowRight")`; return `{ reachedX: x, triggered: <the challenge-length
    check> }`, logging a console.error if 80 iterations elapsed without reaching either exit
    condition (this is a genuine "mechanic unreachable" finding, not a script bug).

    Define `async function resolveIfBoxed(page, renderChoices)`: if `renderChoices` is `false`
    (the collect zone — no key handlers exist per challenge.js), return `{ resolved: null }`
    immediately without pressing any key (movement stays live per collect.js's design, so the
    outer loop can continue past it unresolved). If `renderChoices` is `true`, cycle
    `for (const k of ["1","2","3","4"])`: `await page.keyboard.press(k)`,
    `await page.waitForTimeout(200)`, `const left = await page.evaluate(() => get("challenge").length)`;
    break and return `{ resolved: true }` as soon as `left === 0`; if all 4 presses are exhausted
    without reaching 0, return `{ resolved: false }`.

    Main loop: for `i` from 0 to `LEVEL_ORDER.length - 1`: reuse browser-boot.mjs's
    title→select→level sequence (Space to leave title, `page.keyboard.press("ArrowRight")` `i`
    times with 150ms waits to move the select cursor, `Enter`, 1500ms settle); then
    `const level = getLevel(LEVEL_ORDER[i]); const gapRanges = deriveGapRanges(level.geometry);
    const encounters = deriveEncounters(level.geometry);`; for each `encounter` in ascending-x
    order: call `driveToX`, screenshot to
    `` `.planning/phases/21-real-verification-pass-mechanics-sign-off-integrity-perform-/screenshots/${level.id}-${encounter.tag}-${encounter.x}-before.png` ``
    (build this path the same way `screenshot-phase20.mjs`'s `OUT()` helper does, adapted to this
    phase's directory), call `resolveIfBoxed`, screenshot the `-after.png` variant, push
    `{ level: level.id, tag: encounter.tag, x: encounter.x, triggered, resolved }` onto a
    top-level `results` array; after all of a level's encounters, `page.keyboard.press("Escape")`
    + 800ms wait before moving to the next level (mirroring browser-boot.mjs).

    After all 4 levels: `console.log(JSON.stringify(results, null, 2))`; then print
    `"AUDIT: ALL MECHANICS RESOLVED"` if every `renderChoices:true` entry has `resolved === true`
    AND every entry has `triggered === true`, otherwise print `"AUDIT: FAILURES DETECTED"`
    followed by the JSON-stringified subset of failing entries.

    Wrap the entire per-level driving loop (everything after the browser/context launch) in
    `try { ... } finally { await context.close(); await browser.close(); server.close(); }` —
    the identical cleanup idiom already used by all 3 existing scripts (T-21-02). End with
    `process.exit(0)` unconditionally: this script is a diagnostic tool whose console output
    Task 2 interprets, not a pass/fail commit gate (unlike browser-boot.mjs).
  </action>

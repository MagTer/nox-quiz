<interfaces>
scripts/lib/mechanic-drive.mjs (created by Plan 21-05 — a dependency of this plan) exports:
  deriveEncounters(geometry)                       — ascending-x-sorted { x, tag, renderChoices }[]
  driveToXClimbing(page, targetX, opts)             — async, returns { reachedX, triggered }
  resolveIfBoxed(page, renderChoices)                — async, returns { resolved }

src/levels/index.js exports getLevel(id) and LEVEL_ORDER — already imported by
scripts/browser-boot.mjs today (LEVEL_ORDER only); this plan additionally imports getLevel.

Per-level first-resolvable-mechanic rule (this plan's own generalization of Plan 21-02's
level-01 precedent, which happened to cover "collect-zone then math-gate" because that is
level-01's own first two encounters): walk a level's `deriveEncounters(...)` list in ascending
order; for every `renderChoices:false` (collect-zone) encounter reached along the way, assert
ONLY `triggered` (it has no key handlers to resolve, by challenge.js's own design) and continue
driving to the NEXT encounter; the moment a `renderChoices:true` encounter (door/math-gate/enemy)
is reached, assert BOTH `triggered` and (via `resolveIfBoxed`) `resolved`, then STOP walking
further encounters for that level. This generalizes correctly across all 4 levels regardless of
whether a level has a collect zone first (level-01/03/04) or not (level-02).

**1. [Rule 1 - Bug] Label+arithmetic concatenation overflowed the canvas for the enemy encounter**
- **Found during:** Task 2 (post-fix visual verification)
- **Issue:** Task 1's initial implementation built `display` as a single concatenated string (`` `${label} ${q.a} × ${q.b}` ``). - **Fix:** Changed `challenge.js`'s display block to render `label` (when present, `prompt` absent) as its own smaller line above a second full-size line holding the bare arithmetic expression, instead of one concatenated string.
- **Files modified:** `src/ui/challenge.js`
- **Verification:** Re-ran the manual teleport-assisted verification script; the regenerated screenshot shows both lines fully legible and within panel bounds. Re-ran the full static gate suite and the shared interactive audit; both remain green with zero regressions.

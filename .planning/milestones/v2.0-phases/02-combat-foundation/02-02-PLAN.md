  <action>
    Insert MODULE 9 (CombatEngine) as a closure-based IIFE immediately after `window.DungeonState = DungeonState;`, before DOMContentLoaded.

    Private state:
    - `let currentFloor = null;`
    - `let floorDef = null;` (the FloorConfig definition for the active floor)

    Public API:

    `startCombat(floorNumber)`: Calls `FloorConfig.getFloor(floorNumber)` and stores the result as floorDef. Sets currentFloor = floorNumber. Calls `DungeonState.setEnemyHP(floorDef.hp)` to initialize enemy HP. Calls `GameFSM.transition('COMBAT')` to advance the FSM. Returns the object: `{ enemy: floorDef.enemy, emoji: floorDef.emoji, enemyHP: floorDef.hp, playerHP: DungeonState.get().playerHP }`.

    `resolveAnswer(isCorrect)`: The core combat tick. All damage values from CONFIG.DUNGEON constants (no magic numbers). Returns a result object (shape defined below).

    If `isCorrect` is true:
    - Compute newEnemyHP = Math.max(0, DungeonState.get().enemyHP - CONFIG.DUNGEON.DAMAGE_CORRECT)
    - Call DungeonState.setEnemyHP(newEnemyHP)
    - If newEnemyHP === 0 (enemy killed):
      - Call PlayerState.addXp(floorDef.xpReward) — store return value as leveledUp boolean
      - Call GameFSM.transition('LOOT')
      - Return { enemyHP: 0, playerHP: DungeonState.get().playerHP, killed: true, died: false, leveledUp }
    - If newEnemyHP > 0:
      - Return { enemyHP: newEnemyHP, playerHP: DungeonState.get().playerHP, killed: false, died: false, leveledUp: false }

    If `isCorrect` is false:
    - Compute newPlayerHP = Math.max(0, DungeonState.get().playerHP - CONFIG.DUNGEON.DAMAGE_WRONG)
    - Call DungeonState.setPlayerHP(newPlayerHP)
    - If newPlayerHP === 0 (player died):
      - Call GameFSM.transition('DEAD')
      - Return { enemyHP: DungeonState.get().enemyHP, playerHP: 0, killed: false, died: true, leveledUp: false }
    - If newPlayerHP > 0:
      - Return { enemyHP: DungeonState.get().enemyHP, playerHP: newPlayerHP, killed: false, died: false, leveledUp: false }

    `getState()`: Returns `{ floor: currentFloor, floorDef, enemyHP: DungeonState.get().enemyHP, playerHP: DungeonState.get().playerHP }`.

    DIFF-02 note: CombatEngine itself does not select questions — that remains QuestionSelector's job in Phase 3. The FloorConfig tablePools array is available on the floorDef for Phase 3 to use as the question pool filter. Do not add question selection logic to CombatEngine.

    After the IIFE, expose: `window.CombatEngine = CombatEngine;`

    Add section comment: `/* MODULE 9: CombatEngine — Combat resolution and HP math (Phase 2) */`
  </action>

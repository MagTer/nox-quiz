    Replace the entire QuestionSelector module with the full weighted implementation from RESEARCH.md Pattern 3. Use the Edit tool to replace the existing QuestionSelector IIFE. The replacement must implement all four internal functions (calculateWeights, weightedRandom, generateDistractors, shuffle) and a single public `selectNext(playerState)` method.

    **calculateWeights(playerState)** — exact algorithm per CONTEXT.md locked decisions:
    - For each table in CONFIG.HARD_TABLES (6,7,8,9):
      - `acc = playerState.getAccuracy(table)`
      - `w = Math.pow(1 - acc, 1.5)`
      - If `acc < CONFIG.STRUGGLE_THRESHOLD (0.60)`: `w *= CONFIG.STRUGGLE_BOOST (1.5)`
      - If `playerState.isMastered(table)`: `w *= 0.3` (reduce weight for mastered tables, but never 0 — they can still appear)
      - `weights[table] = w`
    - For each table in CONFIG.EASY_TABLES (1,2,3,4,5):
      - `acc = playerState.getAccuracy(table)`
      - `weights[table] = Math.pow(1 - acc, 0.8) * 0.3`
      - If mastered: `weights[table] *= 0.3`
    - Return weights object
    - Edge case: if total weight is 0 (all tables mastered), reset all weights to equal 1.0 per table so the game never stalls

    **weightedRandom(weights)** — standard weighted roll:
    - `total = Object.values(weights).reduce((a, b) => a + b, 0)`
    - `roll = Math.random() * total`
    - Iterate over `Object.entries(weights)`: subtract each weight from roll; if roll <= 0 return `parseInt(table, 10)`
    - Fallback: return `CONFIG.HARD_TABLES[0]` (should never reach this with proper weights)

    **generateDistractors(answer, table, multiplicand)** — per CONTEXT.md locked decision (±1 ±2 same multiplier):
    - Create a Set of candidates from `[-2, -1, 1, 2].map(d => table * (multiplicand + d))`
    - Filter: positive values only, not equal to answer
    - Add one wrong-table distractor: pick a hard table that is not the current table, compute `wrongTable * multiplicand`, add if positive and not answer
    - Convert Set to array. If fewer than 3 unique candidates exist, add adjacent values (table * (multiplicand + 3), table * (multiplicand - 3)) until 3 are available
    - Fisher-Yates shuffle the pool, then pick the first 3
    - Return array of exactly 3 distractors (never fewer — if somehow still short, pad with `answer + Math.floor(Math.random() * 5) + 1` as last resort, filtered for uniqueness)

    **shuffle(arr)** — Fisher-Yates (Knuth) implementation (in-place, returns shuffled array):
    ```
    for (let i = arr.length - 1; i > 0; i--) {
      const j = Math.floor(Math.random() * (i + 1));
      [arr[i], arr[j]] = [arr[j], arr[i]];
    }
    return arr;
    ```
    This replaces any naive `sort(() => Math.random() - 0.5)` — the naive sort has statistical position bias. Comment: `// Fisher-Yates uniform shuffle — not sort(random) which is biased`

    **selectNext(playerState)** — public API:
    - `weights = calculateWeights(playerState)`
    - `table = weightedRandom(weights)`
    - `multiplicand = Math.floor(Math.random() * 10) + 1` (range 1–10, locked decision)
    - `answer = table * multiplicand`
    - `distractors = generateDistractors(answer, table, multiplicand)`
    - `options = shuffle([answer, ...distractors])`
    - Return `{ table, multiplicand, answer, options, question: table + ' × ' + multiplicand }`
    - Note: use string concatenation for question, not template literal (avoids any future accidental HTML injection risk in adjacent code)

    **Update App.nextQuestion()** to pass PlayerState to selectNext:
    Replace `QuestionSelector.selectNext()` with `QuestionSelector.selectNext(PlayerState)`.

    **Remove the naive comment** (`// Weighted selection by accuracy added in Plan 03`) from the new implementation.

    Do not change any other module. The only changes are: QuestionSelector module (full replacement) and App.nextQuestion() call (add PlayerState argument).

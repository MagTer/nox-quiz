    **Full PersistenceStore implementation:**

    The module is an IIFE returning a public API. Internal constants:
    - `KEY = CONFIG.SAVE_KEY` (reads from CONFIG — do not hardcode the key string again)
    - `VERSION = 1`

    `defaults()` — returns a fresh state object:
    ```
    { version: VERSION, xp: 0, level: 1,
      accuracy: { 1:0.5, 2:0.5, 3:0.5, 4:0.5, 5:0.5, 6:0.4, 7:0.4, 8:0.4, 9:0.4 },
      history: {} }
    ```
    Note: accuracy keys must be numbers. When JSON roundtrips, keys become strings. PlayerState.fromJSON must handle string keys (e.g., `accuracy['6']` instead of `accuracy[6]`). Address this in fromJSON by iterating over the parsed accuracy object and assigning to numeric keys.

    `load()`:
    - Wrap entire body in try/catch
    - `localStorage.getItem(KEY)` — if null, return defaults()
    - `JSON.parse(raw)` — if throws SyntaxError, return defaults() and log warning
    - Check `data.version === VERSION`; if not, log "Save version mismatch, using defaults" and return defaults() (forward-compat stub — future migration hook)
    - Validate top-level keys: `data.xp` must be a finite number ≥ 0, `data.level` must be integer ≥ 1. If either fails validation, return defaults() with a console.warn.
    - Return the validated data object
    - On any other caught error, log and return defaults()

    `save(playerState)`:
    - Wrap entire body in try/catch
    - `JSON.stringify({ version: VERSION, ...playerState.toJSON() })`
    - `localStorage.setItem(KEY, serialized)`
    - On catch: if `e.name === 'QuotaExceededError'` log `console.warn('[MathLab] localStorage full — progress may not save')`. For any other error, log `console.warn('[MathLab] Save failed:', e)`.
    - Return void in all cases — never throw to caller

    **Wire save events** (add after App bootstrap inside DOMContentLoaded, or at module scope after DOMContentLoaded wires up):
    ```javascript
    document.addEventListener('visibilitychange', () => {
      if (document.hidden) PersistenceStore.save(PlayerState);
    });
    window.addEventListener('beforeunload', () => {
      PersistenceStore.save(PlayerState);
    });
    ```

    **Update PlayerState.fromJSON** to handle JSON string-keyed accuracy object:
    In the existing `fromJSON(data)` method, when copying `data.accuracy` into the internal `accuracy` object, iterate over the incoming keys and assign with integer conversion:
    ```javascript
    if (data.accuracy && typeof data.accuracy === 'object') {
      Object.entries(data.accuracy).forEach(([k, v]) => {
        const table = parseInt(k, 10);
        if (table >= 1 && table <= 9 && typeof v === 'number') {
          accuracy[table] = v;
        }
      });
    }
    ```
    Similarly handle `history` — only copy integer keys 1–9, only accept arrays of booleans:
    ```javascript
    if (data.history && typeof data.history === 'object') {
      Object.entries(data.history).forEach(([k, v]) => {
        const table = parseInt(k, 10);
        if (table >= 1 && table <= 9 && Array.isArray(v)) {
          history[table] = v.filter(x => typeof x === 'boolean').slice(-CONFIG.MASTERY_WINDOW);
        }
      });
    }
    ```

    **IMPORTANT: Do not change any other module.** Only modify: PersistenceStore (replace stub), PlayerState.fromJSON (add JSON key validation), and add the two event listeners (visibilitychange and beforeunload). The rest of the file is unchanged.

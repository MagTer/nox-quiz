Use IIFE-returning-object (closure module) pattern for all modules — no classes, no globals outside the closures.

    **Module 1: CONFIG**
    A plain `const CONFIG = { ... }` object (not a closure) holding all magic numbers. Required keys and values (all locked from CONTEXT.md):
    - `XP_EASY: 10` — XP for correct answer on tables 1–5
    - `XP_HARD: 20` — XP for correct answer on tables 6–9
    - `BASE_XP: 200` — XP required for Level 1 → Level 2
    - `LEVEL_MULT: 1.3` — per-level threshold multiplier
    - `ADVANCE_DELAY_MS: 1000` — delay before auto-advancing to next question (1s, locked decision)
    - `SEGMENT_COUNT: 20` — number of XP bar segments
    - `HARD_TABLES: [6,7,8,9]`
    - `EASY_TABLES: [1,2,3,4,5]`
    - `SAVE_KEY: 'mathlab_save_v1'` — localStorage key (locked in CONTEXT.md)
    - `ACCURACY_ALPHA: 0.15` — EWMA weight for new answer
    - `MASTERY_THRESHOLD: 0.80` — 80% accuracy over last 10 to reduce drilling
    - `STRUGGLE_THRESHOLD: 0.60` — below this → 1.5× weight boost
    - `STRUGGLE_BOOST: 1.5`
    - `MASTERY_WINDOW: 10` — sliding window size for mastery check

    **Module 2: XpCalculator**
    Closure module. Implements:
    - `getLevelThreshold(level)` → `Math.round(CONFIG.BASE_XP * Math.pow(CONFIG.LEVEL_MULT, level - 1))`
    - `calculateXp(table)` → `CONFIG.HARD_TABLES.includes(table) ? CONFIG.XP_HARD : CONFIG.XP_EASY`

    **Module 3: PlayerState**
    Closure module exactly as shown in RESEARCH.md Pattern 1. Key points:
    - Internal vars: `let xp = 0; let level = 1;`
    - `accuracy` object initialized to `{1:0.5, 2:0.5, 3:0.5, 4:0.5, 5:0.5, 6:0.4, 7:0.4, 8:0.4, 9:0.4}` — hard tables start at 40% (higher weight in selector)
    - `history` object for sliding window: `{}` (table → boolean array of last N answers)
    - `addXp(points)` — adds XP, detects level-up with `xp -= threshold` (NOT `xp = 0`) surplus carryover, returns boolean (true = leveled up)
    - `updateAccuracy(table, isCorrect)` — EWMA: `acc = acc * (1 - CONFIG.ACCURACY_ALPHA) + (isCorrect ? 1 : 0) * CONFIG.ACCURACY_ALPHA`; updates history window (max CONFIG.MASTERY_WINDOW entries)
    - `getAccuracy(table)`, `getLevel()`, `getXp()`, `isMastered(table)`, `toJSON()`, `fromJSON(data)`
    - `fromJSON` validates incoming keys before assigning: only copy `xp`, `level`, `accuracy`, `history` from the parsed object — do not `Object.assign(PlayerState, untrustedData)` (security: RESEARCH.md security section, prototype pollution mitigation)
    - Expose a `reset()` method that reinitializes to defaults (useful for testing)

    **Module 4: PersistenceStore (stub for Plan 01 — full implementation in Plan 02)**
    Closure module with `load()` always returning default state and `save()` being a no-op. This stub allows App.init to call `PersistenceStore.load()` and `PersistenceStore.save()` without errors. Comment: `// Full localStorage implementation added in Plan 02`.

    **Module 5: QuestionSelector (naive random — weighted selection in Plan 03)**
    Closure module. For this plan, use simple uniform random table selection (equal probability for all 9 tables). Implements:
    - `selectNext()` → picks random table from all 9, random multiplicand 1–10, computes answer, generates 3 distractors using the distractor strategy from CONTEXT.md (±1 ±2 of multiplicand on same table, filter `v !== answer && v > 0`), Fisher-Yates shuffles all 4 options.
    - Returns `{ table, multiplicand, answer, options, question }` where `question` is `` `${table} × ${multiplicand}` ``.
    - Fisher-Yates shuffle implementation as shown in RESEARCH.md Pattern 3.
    - Distractor generation: for each of `[-2,-1,1,2]`, compute `table * (multiplicand + d)`, filter positive and not equal to answer, deduplicate. If fewer than 3 distractors available, add `wrongTable * multiplicand` where wrongTable is a hard table ≠ current table. Pick 3 from the pool.
    - Comment: `// Weighted selection by accuracy added in Plan 03`
    - Expose `selectNext` as the only public method.

    **Module 6: DOM cache and Renderer**
    Declare `const DOM = {}` immediately inside `document.addEventListener('DOMContentLoaded', () => { ... })`.
    Populate DOM cache at init:
    ```
    DOM.levelDisplay = document.getElementById('level-display')
    DOM.xpBar = document.getElementById('xp-bar')
    DOM.questionText = document.getElementById('question-text')
    DOM.optionsList = document.getElementById('options-list')
    DOM.levelupOverlay = document.getElementById('levelup-overlay')
    DOM.questionLegend = document.getElementById('question-legend')
    ```
    Never call `getElementById` outside this init block.

    Renderer module (plain object, not a closure — it uses DOM cache directly):
    - `updateHud(xp, level)` — sets `DOM.levelDisplay.textContent = 'Level ' + level`, calls `renderXpBar(xp, level)`
    - `renderXpBar(xp, level)` — clears `DOM.xpBar.innerHTML = ''`, creates CONFIG.SEGMENT_COUNT div elements with class `xp-segment` (+` filled` for the filled proportion); appends to DOM.xpBar. Compute `filledCount = Math.floor((xp / XpCalculator.getLevelThreshold(level)) * CONFIG.SEGMENT_COUNT)`.
    - `showQuestion(questionObj)` — sets `DOM.questionText.textContent = questionObj.question + ' = ?'`, clears `DOM.optionsList`, then builds 4 `<li class="option-item">` elements each containing a visually-hidden `<input type="radio" name="answer" id="opt-N" value="VALUE">` and a `<label for="opt-N">VALUE</label>`. Appends all 4 to options list. Sets `DOM.questionLegend.textContent = 'Choose the correct answer'`.
    - `showLevelUpOverlay(newLevel)` — sets `DOM.levelupOverlay.textContent = 'LEVEL ' + newLevel + '!'`, adds class `active`, listens for `animationend` with `{ once: true }` to remove `active`.
    - `clearFeedback()` — removes `correct`, `wrong`, `disabled` classes from all `.option-item` elements.

    **Module 7: InputHandler**
    Plain object. Has `currentQuestion = null` and `locked = false` (prevents double-click during 1s feedback window).
    - `setup()` — attaches a single click event listener to `DOM.optionsList` (event delegation). On click: if `locked` return. If `event.target.tagName === 'LABEL'` or `event.target.tagName === 'INPUT'`, find the associated value from the label's `for` attribute or input's value, call `InputHandler.handleAnswer(selectedValue)`.
    - `handleAnswer(selectedValue)` — sets `locked = true`. Gets all `.option-item` elements. Adds `disabled` to all. Checks `parseInt(selectedValue, 10) === InputHandler.currentQuestion.answer`. If correct: adds `correct` class to the clicked item, calls `XpCalculator.calculateXp(currentQuestion.table)`, calls `PlayerState.addXp(xp)`, if addXp returns true calls `Renderer.showLevelUpOverlay(PlayerState.getLevel())`. If wrong: adds `wrong` to clicked, adds `correct` to the item whose label value equals the correct answer. Always: calls `PlayerState.updateAccuracy(currentQuestion.table, isCorrect)`, calls `PersistenceStore.save(PlayerState)`, calls `Renderer.updateHud(PlayerState.getXp(), PlayerState.getLevel())`. After `CONFIG.ADVANCE_DELAY_MS` via `setTimeout`, calls `App.nextQuestion()` and sets `locked = false`.
    - IMPORTANT: The `setTimeout` here is the 1-second feedback delay. It must NOT be exposed as any visible UI element. No progress bar drains during this second, no text counts down.

    **App bootstrap (inside DOMContentLoaded)**:
    ```
    const savedData = PersistenceStore.load();
    PlayerState.fromJSON(savedData);
    InputHandler.setup();
    Renderer.updateHud(PlayerState.getXp(), PlayerState.getLevel());
    App.nextQuestion();
    ```

    **App object:**
    - `nextQuestion()` — calls `const q = QuestionSelector.selectNext()`, sets `InputHandler.currentQuestion = q`, calls `Renderer.showQuestion(q)`.

    **Security requirements (from RESEARCH.md security section):**
    - Never use `innerHTML` to set question text or option values — always `textContent`
    - When `PersistenceStore.load()` returns data (Plan 02), `PlayerState.fromJSON()` must only read explicitly named keys (`xp`, `level`, `accuracy`, `history`) — no `Object.assign(this, data)` spread
    - Selected answer value comes from `label.textContent` or `input.value` which are integers generated by the game — validate with `parseInt` before comparing

    The entire JS lives inside `document.addEventListener('DOMContentLoaded', () => { ... })` at the bottom of the body, or in a single `<script>` tag. Module declarations that don't reference DOM (CONFIG, XpCalculator, PlayerState, PersistenceStore, QuestionSelector) can be declared before the DOMContentLoaded handler. DOM cache and Renderer, InputHandler, App must be inside the DOMContentLoaded callback.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → PlayerState.fromJSON | Saved JSON is read back from browser storage; could be tampered by the user via DevTools |
| User click → InputHandler.handleAnswer | Radio button / label click; selected value is an integer from the generated options array |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | PlayerState.fromJSON reads localStorage JSON | mitigate | Only copy explicitly named keys (xp, level, accuracy, history) from parsed object; never Object.assign(this, data) spread; validate types (xp must be a number ≥ 0, level ≥ 1) before assignment |
| T-01-02 | Tampering | User edits XP/level via DevTools localStorage | accept | Single-device solo game with no leaderboard or external claims; self-cheating has no impact beyond the user's own session |
| T-01-03 | Tampering | XSS via innerHTML if question text is ever rendered with innerHTML | mitigate | All question content (question text, option values) must use textContent exclusively; this is enforced in Renderer.showQuestion — task action explicitly forbids innerHTML for content |
| T-01-04 | Information Disclosure | Answer in source code | accept | Correct answer is computed client-side by design; multiplicand randomization means no pattern to memorize; this is a local learning game |
| T-01-05 | Denial of Service | localStorage QuotaExceededError crashes save loop | mitigate | All localStorage.setItem calls wrapped in try/catch; QuotaExceededError caught and logged to console; game continues in memory (implemented in Plan 02 PersistenceStore) |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | Zero external packages in this phase or any phase; no package manager interaction occurs |
</threat_model>

# Milestone Audit: Math Lab v2.0 — Dungeon Crawler Phases

**Milestone:** v2.0 — Dungeon Crawler Phases
**Audited:** 2026-06-22 (re-audit after DIFF-01 fix, commit 37969d7)
**Status:** gaps_found
**Score:** 26/27 requirements satisfied

## Executive Summary

The DIFF-01 gap from the previous audit (floor table pool gating) has been fixed — `QuestionSelector.selectNext()` now accepts an `allowedTables` parameter, and `CombatInputHandler.nextQuestion()` passes `CombatEngine.getState().floorDef.tablePools` to gate questions to the floor's specific table pool. The fix is confirmed in code (lines 1892–1893) and verified by the integration checker.

However, a related gap persists: **DIFF-02 (EWMA accuracy weighting during dungeon play) is unsatisfied.** `PlayerState.updateAccuracy()` is called in quiz mode (`InputHandler.handleAnswer`, line 1804) but is absent from `CombatInputHandler.handleAnswer()` (lines 1839–1889). All dungeon combat answers silently bypass the adaptive weighting system — per-table accuracy weights are frozen at dungeon-entry state for the duration of any run. The fix is a single line: `PlayerState.updateAccuracy(q.table, isCorrect)` inside `CombatInputHandler.handleAnswer()` after line 1846, matching the quiz-mode pattern.

The dungeon is fully playable end-to-end. 5 of 6 E2E flows work correctly. The ADHD safety checklist passed 6/6 automated checks. Floor-gated question selection (DIFF-01) now works correctly.

**Estimated scope:** 1–2 lines in `math-lab.html`. No structural changes required.

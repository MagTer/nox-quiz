# Milestone Audit: Math Lab v2.0 — Dungeon Crawler Phases

**Milestone:** v2.0 — Dungeon Crawler Phases
**Audited:** 2026-06-22
**Status:** gaps_found
**Score:** 25/27 requirements satisfied

## Executive Summary

The v2.0 Dungeon Crawler milestone is substantially complete — 5 of 6 E2E user flows work end-to-end, the ADHD safety checklist passed all 6 automated checks, and the dungeon is playable from start to finish with combat, loot, floor progression, and death/retry working correctly. However, one critical requirement is unimplemented:

**DIFF-01 (floor table pool gating) is not enforced in question selection.** The `tablePools` data is defined in `FloorConfig` but `QuestionSelector.selectNext()` never reads it — players on Floor 1 (×2×3×5) receive questions from all 9 multiplication tables instead of only from `[2, 3, 5]`. This is a design-level gap that undermines the core educational intent of floor-gated difficulty progression.

## Gap Closure Plan

### Critical: Close DIFF-01 / DIFF-02

**Change required:** `QuestionSelector.selectNext(playerState, allowedTables?)` — add optional second parameter. When `allowedTables` is provided, `calculateWeights()` must zero-out weights for tables outside the allowed set before the weighted-random draw.

**Caller change:** `CombatInputHandler.nextQuestion()` (lines 1888 and 1927) must retrieve the current floor's pool and pass it:
```js
const pool = CombatEngine.getState().floorDef.tablePools;
const q = QuestionSelector.selectNext(PlayerState, pool);
```

**Estimated scope:** 2 functions in `math-lab.html`; ~15 lines of change.

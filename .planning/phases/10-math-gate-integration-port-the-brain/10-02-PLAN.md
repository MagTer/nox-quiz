<objective>
Create the single bridge module `src/ui/mathGate.js` — the ONLY connector between the Kaplay scene and the pure brain. It exports `openMathGate({ brain, onClear })`, which renders an in-world, camera-immune (`fixed()`), dimmed gate overlay (GATE-01): a full-screen dim layer + a dark-grunge panel + the big question expression + four answer boxes. Input is dual: number keys 1-4 AND mouse-click. A wrong answer is forgiving — mark/shake the box, keep the SAME question, gate stays open, no penalty, no game-over (GATE-04). A correct answer flashes + shows LEVEL CLEAR, cleanly closes the gate (cancel key controllers + destroy the tagged subtree), and invokes `onClear()` exactly once. There is NO timer/countdown anywhere (GATE-05).

Purpose: This is the in-world gate the kid sees and interacts with — it must read as part of the game (not a system quiz popup), be ADHD-safe (forgiving, no time pressure), and be leak-safe across replays (the highest-risk area per research).

Output: `src/ui/mathGate.js` (new bridge + UI module). The scene wiring that calls it is Plan 03.
</objective>

Build the display string here (the brain dropped its `question:` field). The multiplication glyph is U+00D7; if it renders as tofu in the first playtest, fall back to `'x'`.

    3. Four answer boxes from `q.choices`, laid out below the question. Each box is an `area()` object tagged `"math-gate"` carrying its index, with its label text. Wire BOTH input paths to a single `choose(i)` function:
       - Mouse: `box.onClick(() => choose(i))` — object-scoped, auto-cleaned on destroy.
       - Keys: capture EVERY controller — `const keyCtrls = ["1","2","3","4"].map((k, i) => onKeyPress(k, () => choose(i)));` — these are GLOBAL handlers and MUST be cancelled on close (Pitfall 1).

    4. `function choose(i)`: `const picked = q.choices[i]; const correct = picked === q.answer; brain.reportResult(q.a, correct);` (push result, gate to brain, one-way). Then branch:
       - WRONG (GATE-04 forgiving): mark/shake the chosen box (e.g. `shake(...)` and/or flash the box color to a red accent), KEEP the same `q`, leave the gate open and input live. Do NOT mutate any run-ending state, do NOT call `onClear`, do NOT close. The kid can pick again immediately.
       - CORRECT (GATE-03 hook — celebration): flash the chosen box to a neon-green accent, show a "LEVEL CLEAR" `text()` (fixed, tagged `"math-gate"`), then `close()` and call `onClear()` EXACTLY once (guard against double-fire with a boolean latch). Keep onClear simple here — the scene supplies what "clear" means.

    5. `function close()`: `keyCtrls.forEach((c) => c.cancel()); destroy("math-gate");` — cancel every global key handler (no leak) AND destroy every tagged gate object (boxes/panel/dim/text). Both calls are REQUIRED.

    HARD CONSTRAINTS (encode as code; verified by greps):
    - NO timer/time-pressure: the gate stays open until a correct answer (GATE-05/SAFE-01). Do not schedule any deferred auto-advance or auto-close. A celebration flash via `tween`/color-set is fine ONLY if it does not gate progression on elapsed time.

<objective>
Replace Phase 18's procedurally-generated placeholder `assets/player.png` and
`assets/tiles/ground.png` with real, curated, licensed CC0 pixel art, through Phase 18's
unchanged technical contract (`18-UI-SPEC.md`) — no src/ code changes. Source real Kenney art,
build it through a new reproducible pipeline (`scripts/build-art-assets.py`), and record
matching CC0 license proof.

Purpose: ART-05 and ART-06 both fail today because `scripts/generate-art-assets.py` draws flat
rectangles + random noise instead of real art (confirmed by `git log --stat` this session), while
`CREDITS.md`/`assets/LICENSES/player.txt`/`ground.txt` still falsely claim the shipped PNGs are
genuine HorusKDI CC0 crops. This plan fixes both the asset content and the license record.

**Important correction to CONTEXT.md's Area 1 (surfaced explicitly, not buried):** CONTEXT.md
locked "source the player from Kenney's Pixel Platformer pack because it ships walk/jump
animation frames." Direct inspection this session (20-RESEARCH.md Assumptions Log A1, Open
Question 1) found that claim factually wrong — Pixel Platformer's character tiles are static
24x24 blob mascots with no walk cycle, the same single-static-figure problem CONTEXT.md correctly
flagged in the *previous* dungeon pack. This plan instead sources the player from Kenney
**"Platformer Characters" (Adventurer pose set)**, a different, equally CC0, same-publisher pack
that genuinely ships named idle/stand/walk1/walk2/jump poses. "Pixel Platformer" is retained
exactly as CONTEXT.md specified, but only for the ground/tileset. This substitution keeps the
*intent* of Area 1 (real animated player, not a static figure) while correcting the specific pack
name.

Output: `scripts/build-art-assets.py` (new build pipeline), `assets/player.png` +
`assets/tiles/ground.png` (real content, unchanged dimensions), rewritten
`assets/LICENSES/player.txt` + `ground.txt`, updated `CREDITS.md` rows.

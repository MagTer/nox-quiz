  <name>Task 2: Build scripts/check-contrast.mjs (WCAG ratio + banned-hue guardrail)</name>
  <files>scripts/check-contrast.mjs</files>
  <read_first>26-RESEARCH.md "WCAG relative-luminance + contrast-ratio formula" code example; src/config.js's finished CONFIG.PALETTE; scripts/validate-levels.mjs (structural precedent for a plain-Node CLI script with a clear pass/fail exit code, no framework)</read_first>
  <behavior>
    - relativeLuminance([r,g,b]) implements the standard W3C sRGB-to-linear-then-luma formula (threshold 0.03928, gamma 2.4).
    - contrastRatio(a,b) returns (lighter+0.05)/(darker+0.05).
    - A "role pairing" table defines which PALETTE role renders on which background, with a threshold: TEXT-on-SURFACE (4.5), TEXT-on-BG (4.5), TEXT_DIM-on-BG (4.5), REWARD-on-BG (4.5, used as text in title/hud/mathGate), DANGER-on-BG (4.5, used as text in title/challenge), CLEARED-on-SURFACE_UNLOCKED (3.0, UI outline), MUTED_BORDER-on-BG (3.0, UI outline), BORDER-on-BG (3.0, UI outline), ACCENT_MOSS/ACCENT_SLATE/ACCENT_RUST-on-BG (3.0, UI-component/background-tint role, not text).
    - isBannedHue([r,g,b]) returns true iff BOTH (r - g) > 20 AND (b - g) > 20 (green channel meaningfully suppressed relative to both red and blue — this is what reads as magenta/pink/mauve regardless of saturation, per 26-CONTEXT.md's explicit "desaturated magenta/mauve reads as pink" concern). Applied to every CONFIG.PALETTE value.
    - Script prints a table (role, pairing, ratio, threshold, pass/fail) to stdout, then exits 1 if ANY pairing fails its threshold OR any color trips isBannedHue; exits 0 and prints "contrast checks: PASS" otherwise.
  </behavior>
  <action>
Create scripts/check-contrast.mjs as a plain Node ESM script (no dependency — matches the project's zero-new-dependency posture) that imports `{ CONFIG }` from `../src/config.js`, implements the behavior above, and is runnable as `node scripts/check-contrast.mjs` from the repo root. Follow validate-levels.mjs's shape: a CLI script with a clear numeric exit code, not a test framework. Do not hardcode a copy of the palette hex values — read them live from `CONFIG.PALETTE` so the script can never drift from the real source of truth.
  </action>
  <verify>
    <automated>node scripts/check-contrast.mjs; echo "exit:$?"</automated>
  </verify>
  <done>scripts/check-contrast.mjs exists, imports CONFIG.PALETTE live, and running it prints a role/ratio/pass-fail table ending in "contrast checks: PASS" with exit code 0 against the current palette.</done>

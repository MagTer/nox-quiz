<task type="auto">
  <name>Task 1: Write nginx.conf with the .mjs MIME fix and Dockerfile</name>
  <files>docker/nginx.conf, docker/Dockerfile</files>
  <read_first>
    - .planning/phases/07-project-setup-deployment/07-RESEARCH.md (Pattern 1 nginx .mjs MIME fix, Pattern 4 single-stage Dockerfile, Pitfall 4 types{} drops js, Security Domain optional hardening)
  </read_first>
  <action>
    Write `docker/nginx.conf` as a server block listening on port 80, root `/usr/share/nginx/html`, index `index.html`. The critical content: after `include /etc/nginx/mime.types;`, add a `types { application/javascript js mjs; }` block that maps BOTH `js` AND `mjs` to a JavaScript MIME type. Re-declaring `js` alongside `mjs` is required because a `types{}` block replaces (does not merge with) the inherited mapping for the listed extensions — omitting `js` would regress `.js` to octet-stream (Pitfall 4). Set `default_type application/octet-stream;`. Use a `location /` with `try_files $uri $uri/ /index.html;`. Do NOT enable autoindex (no directory listing). Add the low-effort hardening headers per the Security Domain: `add_header X-Content-Type-Options "nosniff" always;` and `server_tokens off;`.

    Write `docker/Dockerfile` as a single-stage image from `nginx:alpine` (no build step). Replace the default site config with ours: `COPY docker/nginx.conf /etc/nginx/conf.d/default.conf`. Copy ONLY the static set, flattening `src/` to the web root for clean prod URLs (Option B): `COPY src/ /usr/share/nginx/html/`, `COPY lib/ /usr/share/nginx/html/lib/`, `COPY assets/ /usr/share/nginx/html/assets/`. Do NOT copy `.git`, `archive/`, `.planning/`, or the Dockerfile/nginx.conf themselves into the web root (info-disclosure mitigation T-07-02). `EXPOSE 80`. Do not override CMD (nginx:alpine already runs nginx in the foreground).

    The build context is the repo root; the image is built with `-f docker/Dockerfile` and context `.` so the `COPY src/`/`lib/`/`assets/` paths resolve (Pitfall 5). The literal `application/javascript js mjs` MUST appear in nginx.conf and `nginx:alpine` MUST appear in the Dockerfile.
  </action>
  <verify>
    <automated>grep -q "application/javascript js mjs" docker/nginx.conf && grep -q "nginx:alpine" docker/Dockerfile && grep -q "COPY docker/nginx.conf" docker/Dockerfile && grep -qE "COPY src" docker/Dockerfile && grep -qE "COPY lib" docker/Dockerfile && ! grep -qi "autoindex on" docker/nginx.conf</automated>
  </verify>
  <acceptance_criteria>
    - `docker/nginx.conf` includes the base mime.types and a `types { application/javascript js mjs; }` block (both `js` and `mjs` present), `try_files` fallback, no autoindex, and the nosniff/server_tokens hardening.
    - `docker/Dockerfile` is single-stage `nginx:alpine`, copies the custom nginx.conf to the conf.d default, and copies only `src/`, `lib/`, `assets/` into the web root (src flattened to root).
    - No source-control or planning/archive files are copied into the image.
  </acceptance_criteria>
  <done>The container config exists with the .mjs MIME fix and a minimal hardened static-serving setup; only intended files reach the image.</done>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internet → nginx container | Public, unauthenticated HTTP(S) requests for static files; the only listening port is 80. |
| repo filesystem → image | Docker COPY decides which repo files become publicly served; must copy only the intended static set. |
| Dokploy platform → container | Dokploy terminates TLS and proxies to container port 80 (deferred; config only). |

(No backend, no database, no accounts, no user input, no secrets — proportionate to a static nginx file server.)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02 | Information disclosure | Files copied into the image | mitigate | Dockerfile copies ONLY `src/ lib/ assets/`; never `.git`, `archive/`, `.planning/`, or the config files into the web root. No autoindex (no directory listing). |
| T-07-03 | Tampering/Spoofing | MIME confusion / content sniffing | mitigate | Explicit `.mjs -> application/javascript` mapping + `default_type application/octet-stream` + `X-Content-Type-Options: nosniff`. |
| T-07-04 | Information disclosure | nginx version banner | mitigate | `server_tokens off;` in nginx.conf. |
| T-07-05 | Tampering | Base image drift | accept | `nginx:alpine` resolves to current stable-alpine; acceptable for a static site. May pin `nginx:stable-alpine` if a frozen line is wanted (noted, low risk). |
| T-07-06 | Information disclosure | Missing TLS in production | mitigate (deferred) | `docs/DEPLOY.md` step 6 enables HTTPS/Let's Encrypt via Dokploy on the live deploy. |

# Phase 34.6.1 Plan 01: Wrong-Answer Anti-Mash Settle Summary

**Amplified wrong-answer feedback (stronger shake + full-panel red pulse) plus a closure-local, tween-gated `settling` flag in the shared challenge overlay that makes both mouse and 1-4-key input inert for ~250ms after a wrong pick — defeating input-mashing without any punishment (SAFE-01).**

## Accomplishments
- Added four new `CONFIG.GATE.WRONG_*` tunables (settle duration, peak flash opacity, shake magnitude, flash z-order) — no magic numbers left in `challenge.js`.
- Implemented the anti-mash settle in `src/ui/challenge.js`: a closure-local `settling` flag, set at the top of the wrong-answer branch and guarding the single `choose(i)` funnel used by both `box.onClick` and the `onKeyPress("1".."4")` handlers.
- Wired the settle's lifecycle entirely off a Kaplay `tween(...).onEnd()` callback — the exact same fire-once-cleared-by-tween idiom already proven in `src/mechanics/key.js`'s `hintShowing`. No `setTimeout`/`setInterval`/`wait(`/`loop(`/`lifespan(` anywhere.
- Confirmed the challenge can never soft-lock: `settling` always returns to `false` in the tween's `onEnd()`, the same question stays open/answerable, and a correct pick still fires `onSuccess` exactly once via the untouched `cleared` latch and `close()` path.

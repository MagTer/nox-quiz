## Implementation Decisions

- **D-01 — Wrong-answer feedback (challenge.js): flash + brief tween-gated settle (anti-mash).**
`src/ui/challenge.js` already reddens the picked box, plays the soft "wrong" SFX, `shake(6)`s, and re-asks the SAME question — but because `choose(i)` resolves instantly on every keypress, the player can mash 1‑2‑3‑4 and the correct one closes the panel before a wrong pick registers.

**Locked fix:**
- **Amplify the existing feedback** on a wrong pick: a brief RED panel/dim flash pulse + a stronger `shake` (keep the box-reddening and soft "wrong" SFX that already exist).
- **Add a short input "settle"** (~0.25s, tunable in config) during which BOTH input paths (mouse-click `box.onClick` AND the 1‑4 `onKeyPress` handlers) are ignored, so mashing all four keys no longer slips a wrong answer past. The settle is driven by the flash **tween's `onEnd`** clearing a closure-local `settling` flag — NEVER a timer (SAFE-01 / `check-safety.sh`).
- **Still fully non-punishing:** same question stays open and is re-asked, nothing counts down, no lost progress, no game-over. A correct pick still fires `onSuccess` exactly once and closes cleanly.
- The `settling` flag is **closure-local** (never module-level `let`) — same anti-leak discipline as the file's existing `cleared`/`keyCtrls` handling.

- **D-02 — Coin sprite: crisp CC0 replacement for `assets/coin.png`.**
Replace the low-res `assets/coin.png` (currently 256×32 = 8-frame gold spin, loaded in `src/main.js` with `sliceX: 8`) with a crisp, **CC0-curated** coin sprite in the dark-grunge palette (gold, no pink).
- Preserve the spinning animation. Either match the existing 8-frame sheet geometry OR update the `sliceX`/anim config in `main.js` (+ any config constant) to the new sheet's real dimensions — the loader and the sheet must agree.
- Add the license file under `assets/LICENSES/` and a `CREDITS.md` entry, matching the existing asset-licensing pattern.
- `check-assets-manifest.mjs` must stay green (declared path exists on disk).

- **D-03 — Goal iconography: checkered-flag CC0 sprite (global finish icon).**
Replace `assets/goal.png` (16×16, currently reads as an ambiguous key-ish blob — the root of the "key opens a key" confusion) with a **checkered-flag** CC0 sprite so the finish reads unmistakably as "the end."
- This is a **global** swap — all 8 levels share the `"goal"` sprite, so a flag becomes the finish icon everywhere (desirable: consistent, legible finish).
- The flag sprite MUST be authored to keep the goal's **existing collision footprint** (`CONFIG.GOAL_SIZE = 16`; the `"goal"` entity's `area()`), so level reachability and the validator/browser-boot are unaffected. A visually taller flag is fine only if the collision `area()` stays 16px and does not overlap platforms above.
- Add license + `CREDITS.md` entry.
- Verify all 8 levels' goals still fire (`validate-levels.mjs` + `browser-boot.mjs` green).

- **D-04 — Key pickup iconography + skip framing (even levels 02/04/06/08).**
The math-skip key pickup currently renders as a plain gold **rect (a box)** — `src/levels/build.js` builds it with `rect(CONFIG.KEY.W, CONFIG.KEY.H)` + `CONFIG.KEY.COLOR`, not a sprite.

**Locked fix (iconography-first, per user):**
- Replace the box rect with a **real key CC0 sprite** (`loadSprite("key", …)` in `main.js`; `sprite("key")` in the build.js key builder). Keep the **20px collision footprint** — `CONFIG.KEY.W`/`H` are load-bearing for `scripts/lib/key-lock-check.mjs`, so the `area()` size must not change. `CONFIG.KEY.COLOR` becomes vestigial (keep only if still used for a debug tint, otherwise leave documented).
- **Rework the framing text** so the flow reads as "grab the key → skip the final math," not "collect a key to open a key": reword the pickup popup (`fx.popupText(..., "+ KEY")` in `key.js`) and/or the persistent HUD key indicator (`CONFIG.HUD.KEY_GLYPH = "KEY"`, shown via `hud.showKey()`) toward skip-legible wording. Keep all strings plain ASCII (TOFU-fallback caution — no emoji glyphs).
- Only even levels place keys; odd/keyless levels are unaffected. The key→skip *mechanic* (`heldKeyIds`, `endSkipArmed`, `XP_KEY_SKIP` in `game.js`) is CORRECT and stays unchanged — this decision only changes the sprite + framing text, never the mechanic.

<specifics>
## Specific Ideas

- **Wrong-answer settle mechanism (D-01):** on a wrong pick, set a closure-local `settling = true`; run a short flash tween (e.g. pulse a red overlay / the dim layer's color) whose `.onEnd()` sets `settling = false`. Guard the TOP of `choose(i)` with `if (settling) return;` (in addition to the existing `if (cleared) return;` and bounds guard) so BOTH mouse and 1‑4 keys are inert during the settle. This is the same tween-gated fire-once pattern `key.js`'s hint uses (`hintShowing` cleared by `tween().onEnd()`).
- **Do NOT add a "refuse to answer" or "lock the wrong box" that could soft-lock** — the challenge must always remain resolvable (the file's own note: never strand a frozen player).
- **Goal flag footprint (D-03):** the `"goal"` entity's `area()` defaults to sprite bounds; if the flag sprite is larger than 16×16, pass an explicit `area({ shape: … })` / scale so the collider stays at the current 16px, or the reachability envelope in `docs/LEVEL-DESIGN.md` is violated.
- **Key sprite footprint (D-04):** keep `area()` at `CONFIG.KEY.W/H` (20px). If the CC0 key art is a different pixel size, scale the sprite visually but pin the collider to 20px.
- **Coin sheet contract (D-02):** `main.js` currently declares `sliceX: 8` for a 256×32 sheet (32px frames). A CC0 coin with a different frame count/size requires updating `sliceX` (+ any `anims`/`speed`) to match — loader and sheet must agree or frames render as garbage.
- **CC0 sourcing:** prefer well-known CC0 sets (Kenney.nl is CC0; OpenGameArt CC0-tagged). Coin + key + checkered-flag are all common CC0 icons. Recolor/tune to the grunge palette if needed (document any modification in the license entry).
- **This is kid-facing polish:** the end-state proof is *playing it* — a wrong answer visibly registers and mashing no longer slips through; the coin looks crisp; the finish reads as a flag; the key reads as a key and its purpose ("skip the final math") is legible. "Checks that don't play the game lie" (CLAUDE.md) — plan for interactive/visual verification, not just green greps.
</specifics>

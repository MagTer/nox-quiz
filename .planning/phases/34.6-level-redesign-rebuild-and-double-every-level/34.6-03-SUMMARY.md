# Phase 34.6 Plan 03: Rebuild level-02 (intense/vertical swamp, math-skip key) + two-path audit Summary

**Rebuilt `src/levels/level-02.js` from scratch as a ~2x-length, open-air intense/vertical swamp (goal.x 4200 -> 8180) with three ascent/descent humps — the third carrying an optional math-skip KEY (no physical lock) on its peak — and added `scripts/audit-endgate-key.mjs`, a dynamically-level-discovering in-engine audit that proves the key-conditional end gate holds both ways: key collected clears the level free, key skipped requires the end math as normal.**

## Accomplishments
- Rebuilt `src/levels/level-02.js` entirely: 12 floor runs (all 120px gaps except two 410px spans, each bridged by a 3-tier hump), 10 platforms (all `h: 16`, all off-grid `y`, zero overlapping pairs anywhere), 35 coins, 6 spikes each centered on its floor with ~260px margins from both edges, exactly one door, one enemy, `mathGates: []`, one secret alcove.
- Authored THREE open-air ascent/descent humps (up from level-01's one) to make the "intense and vertical" even-level archetype distinct from level-01's calm intro while staying inside the L1-2 no-ceiling rule (LEVEL-DESIGN §8): two plain humps plus a third whose peak tier carries the optional math-skip key.
- Placed the key on the third hump's peak (a 140px-wide platform, wider than its neighbors) squarely in a walked middle stretch — past the landing zone from the mount hop, before the takeoff for the descent — so a driven player collects it naturally rather than sailing over it mid-arc (the exact 34.5 x:760 defect class).
- Discarded the throwaway 34.5 key+lock pair (`keys:[{x:900}]` + `locks:[{x:3960}]`); the rebuilt level places `geometry.keys` with NO `geometry.locks` at all (the math-skip usage).
- Re-baselined the level-02 golden geometry fixture in `scripts/smoke-progress.mjs`.
- Built `scripts/audit-endgate-key.mjs`: discovers math-skip levels live from the registry (`geometry.keys?.length > 0 && !geometry.locks?.length`), navigates directly via `go("game", { levelId })` (no select-grid keyboard coupling), and proves both end-gate paths in the real running engine.
- Found and fixed a real audit-script defect (not a game bug) via actually running the new audit: `driveToXPlanned`'s default 15000ms stall timeout wasted time waiting out a "stall" that was actually the key path's own success signal (`player.paused = true` freezes the arrival-progress metric the instant `onReachGoal` fires), and the marker check raced against the scene's own fast (400ms) transition to select, which destroys the "gate-cleared" banner it was checking for. Fixed with a shortened `stallMs` override and a two-signal OR (banner tag present OR the select scene's own entities now live).
- `validate-levels.mjs` reports **zero HARD-FAIL for level-02** (all 8 remaining project-wide HARD-FAILs are pre-existing, deferred, and confined to levels 03/07, not yet rebuilt in this phase); `audit-coins.mjs` collects all 35 of level-02's coins; `audit-phase21-mechanics.mjs` shows door/enemy/secret-alcove all `triggered:true resolved:true`; `audit-endgate-key.mjs` PASSes both paths.

## Decisions Made
- Escalated verticality with THREE humps rather than adding overlapping/ceilinged tiers to reach "intense" — keeps level-02 inside the L1-2 §8 band (no ceilings at all) while still reading as meaningfully more vertical than level-01's one hump, per the plan's Reconciliation note (A3 from 34.6-RESEARCH.md).
- The math-skip key's high route shares its gap-crossing stretch with a plain, bare-jumpable 120px ground-route gap (not a separate detour requiring its own bridge) — reinforces that skipping the key costs nothing structural, only the optional reward, matching CONTEXT's "no penalty for not doing the final math" framing.
- `audit-endgate-key.mjs` navigates via `page.evaluate(() => go("game", { levelId }))` rather than keyboard-driving the select-grid cursor — a deliberate departure from `audit-key-lock.mjs`'s keyboard-navigation pattern, chosen specifically because this script's whole design point is NOT hardcoding a level id, and select-grid tile position depends on where a level sits in `LEVEL_ORDER`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] audit-endgate-key.mjs's own Path A check false-failed on its first run**
- **Found during:** Task 2, first `node scripts/audit-endgate-key.mjs` verification run
- **Issue:** Path A reported `arrived=true, challengeOpened=false` (both correct) but `clearMarkerPresent=false`, failing the gate. Root cause: on the key path, `onReachGoal` sets `player.paused = true` the instant it fires (before the heldKeyIds branch even runs), which freezes the player's position — `driveToXPlanned`'s arrival-progress metric then never advances again, so the call doesn't return until its full default 15000ms stall timeout elapses. By then `CONFIG.FX.BURST_MS` (400ms) has long since fired `go("select")`, tearing down the whole game scene — including the "gate-cleared" tagged banner objects the check was looking for.
- **Fix:** Passed a shortened `stallMs: 3000` override to the goal-approach `driveToXPlanned` calls (harmless on the math path, which always exits via its own `triggered` break, never the stall guard), and changed the marker check to accept EITHER the banner tag (`get("gate-cleared").length > 0`, if the check somehow wins the race) OR the resulting select-scene's own entities (`get("select").length > 0`) — both are equally valid proof the shared `clearLevel()` path actually completed.
- **Files modified:** `scripts/audit-endgate-key.mjs` (this is a test/audit script found and fixed within the same task that authors it — not a game-code bug)
- **Verification:** Re-ran `node scripts/audit-endgate-key.mjs` — exit 0, Path A now reports `keyCollected=true, challengeOpened=false, clearMarkerPresent=true (onSelectScene=true)`, Path B reports `challengeOpened=true`; both PASS.
- **Committed in:** `c2e1aae` (part of Task 2's commit)

**2. [Rule 1 - Bug] Acceptance-criteria grep false-positive on a comment**
- **Found during:** Task 1, running the acceptance-criteria greps after the first draft
- **Issue:** `grep -c 'locks:' src/levels/level-02.js` returned 1, not the required 0 — a comment explaining the OMITTED `locks` field literally contained the substring `locks:` (`// locks: OMITTED — ...`).
- **Fix:** Reworded the comment to describe the same fact without the literal `locks:` substring ("NO physical lock geometry anywhere in this descriptor").
- **Files modified:** `src/levels/level-02.js`
- **Verification:** `grep -c 'locks:' src/levels/level-02.js` now returns 0; re-ran `node scripts/smoke-progress.mjs` and `node scripts/validate-levels.mjs` to confirm the comment-only edit changed nothing structural.
- **Committed in:** `392faec` (part of Task 1's commit)

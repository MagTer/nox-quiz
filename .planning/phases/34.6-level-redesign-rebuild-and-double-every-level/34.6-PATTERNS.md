**CRITICAL deltas from the shipped analogs (do NOT copy these forward):**
- `level-02.js` line 32–41 platforms are `h: 24` — the rebuild uses `h: 16` on EVERY platform (see `level-08.js` lines 106–111, the corrected h:16 tiers).
- `level-02.js` lines 88–116 place a THROWAWAY 34.5 key+lock PAIR (`keys:[{x:900…}]` + `locks:[{x:3960…}]`). The rebuild DISCARDS the lock: even levels place `keys` (no `keyId` → shared default) and OMIT `locks` entirely.
- `bounds.right` is used AS-IS on level-02+ (see `level-08.js` lines 43–50 bounds-convention trap comment) — hand-bump it to `goal.x + GOAL_SIZE + ~64–80px` for each doubled level. level-01 alone derives it (game.js lines 109–119).

**Switchback / vertical reference** (`level-08.js` lines 106–111): tiers reverse direction twice, each consecutive pair overlaps in x by ~70px, rises 72–75px (headroom `rise - 16 - 32 ≥ 24`). Per-tier checkpoint on the side the player lands on (lines 174–179). This is the EVEN-level analog for verticality — but per RESEARCH Pitfall 1, level-02 achieves verticality in OPEN AIR (non-overlapping tiers), reserving real ceilings for L3–8.

**Key placement caveat** (`level-02.js` lines 99–115): a ground/route trigger inside a jump arc gets sailed over (the x:760→x:900 Rule-1 fix). Place each even level's key in a WALKED stretch of its high route, past every landing and before the next takeoff.

## Shared Patterns

### Engine-global discipline (a727c13) — ALL touched src/ files
**Source:** `src/mechanics/key.js` lines 23–26 header; `src/ui/mathGate.js` lines 8–11.
Never reference Kaplay globals (`add`, `destroy`, `tween`, `go`, `pos`…) at module top level — only inside exported function bodies, after `kaplay({ global })`. Descriptors import ONLY `../config.js`; `index.js` stays node-importable. Enforced by `check-import-safety.sh`.

### No timers / self-cleaning tween idiom — game.js + key.js
**Source:** `key.js` lines 124–129 (`tween(...).onEnd(() => destroy(...))`); `game.js` lines 270–274 (`clearTransitionTween.onEnd`).
Delayed effects use `tween().onEnd()`, never `setTimeout`/`wait()`/`loop()`/`lifespan()`. The extracted `clearLevel()` carries this idiom unchanged. Enforced by `check-safety.sh`.

### Anti-leak: closure-local run-state + onSceneLeave cancel
**Source:** `game.js` lines 77–86 (`heldKeyIds`, `clearTransitionTween` closure-local) + lines 359, 371–376 (cancel on scene leave); `key.js` lines 58, 81 (`collected`/`opened` Sets closure-local).
Run state is never module-level `let`; app-bus controllers cancelled on leave. Applies to the new branch (reuses existing closure state — adds none).

### Path-traversal + loopback guard — any new audit script
**Source:** `audit-key-lock.mjs` lines 128–135 (CR-02 resolve+clamp-to-ROOT), loopback-only bind.
Copy verbatim into `audit-endgate-key.mjs`; do NOT weaken.

# 34.6-04 SUMMARY — Prototype human sign-off + locked decisions

## Outcome: PROTOTYPE APPROVED (on the third iteration)

The mandated mid-phase `checkpoint:human-verify` gate. A human (the user) played the rebuilt
levels 01 + 02 and signed off — **not rubber-stamped** (standing precedent; auto_advance was
overridden). Approval came only after two rejections that materially raised the quality bar —
which is exactly the rework insurance the prototype-first sequencing exists to buy.

### Iteration history (why this took three rounds)

1. **v1 (Sonnet executors):** levels 01/02 built validator-legal but FLAT — "hardly any
   difference between level 1 and 2, no climbing, no going backwards." Rejected.
2. **Model escalation (user decision):** the failure was creative level-DESIGN judgment, not
   mechanics — Sonnet builds legal-but-flat geometry. Switched level authoring to **Opus
   executor subagents** (user chose this at the checkpoint). Captured as memory
   `level_design_needs_opus`.
3. **v2 (Opus):** level-02 became a real 370px switchback climb, level-01 gentler-with-character.
   Still rejected: "too simple, repetitive, needs more action, more vertical in both, level 2 even
   higher with options on what path to take." The user flagged the ROOT cause — the requirements
   under-specified ambition.
4. **Raised the bar in the design rules:** added **docs/LEVEL-DESIGN.md §8.5** ("Level ambition")
   — verticality + branching/route-choice + action in EVERY level, level-1-forgiving-but-not-flat,
   open-air, and the fall-stakes calibration (see A-decisions). This drives ALL 8 levels.
5. **v3 (Opus, to §8.5):** APPROVED.
   - level-01 "The First Ascent": mandatory ~196px arch + descent + a visible low/high fork; still
     the gentlest (comfortable margins, 14 checkpoints).
   - level-02 "The Rusted Spire": 518px climb (592px to the key), a diamond fork giving two visible
     routes up that rejoin, 3 switchbacks, interleaved descents, real fall-pits behind 27
     checkpoints, key on the hardest branch.

### Locked decisions

- **A1 — key-skip XP:** CONFIRMED at `CONFIG.PROGRESS.XP_KEY_SKIP = 20` (full credit = a hard-table
  answer). No change.
- **A3 — level-02 verticality / ceilings:** CONFIRMED **open-air** (no enclosed ceilings/tunnels;
  climbs are open stacked tiers, always legible). Recorded in LEVEL-DESIGN.md §8 + §8.5 rule 6. §8
  ramp NOT amended for ceilings.
- **NEW — fall-stakes (agreed this checkpoint):** missed jumps MAY drop into a pit and respawn —
  real platforming tension is wanted — because the respawn is close. NOT punishment (no game-over,
  no lost progress, no timer). Guardrail = generous checkpoint cadence. Recorded in §8.5 rule 4.
- **NEW — the §8.5 raised bar:** every level vertical + multi-route + action; forgiving ≠ flat.
  This is the authoring bar levels 03–08 are built to.

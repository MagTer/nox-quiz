<objective>
Build the STATIC softlock proof for the key/lock mechanic — a pure-data, node-importable validator check that HARD-proves, per level, that every lock's key is reachable from spawn AND on the spawn-side of its lock — plus the shared config tunables both the validator and the (later) mechanic consume, and a RED-first fixture proving the check actually fires.

This is Wave 1 and lands FIRST, before any mechanic code or any level uses a key-lock. It mirrors the milestone's RED-first discipline (Phase 30 mover-validation, Phase 34 headroom gate): the softlock validator is proven to fail RED on a broken fixture BEFORE the mechanic that could produce a softlock exists.

Purpose: KEY-02 requires that NO SOFTLOCK IS POSSIBLE — a key she can miss on a route she cannot re-traverse is a dead end, the worst failure a no-punishment game for a 12-year-old can ship. This plan is the static half of that proof.
Output: `scripts/lib/key-lock-check.mjs` (+ self-test), `scripts/fixtures/bad-level-key.js`, the `LOCK`/`KEY`/`HUD.KEY_*` config tunables, two new exports on `reachability.mjs`, and the composition into `validate-levels.mjs`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| keyboard → game state | The only input crossing into the game is keyboard events read via Kaplay input globals. No untrusted strings, no network, no storage this phase. |
| go() seed data → scene | `go("game", {startX, startY})` payload — author-controlled constants, not user input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | `go("game", data)` seed payload | accept | Data is author-controlled constants; default-guard fields (`data?.startX ?? 64`). No user-supplied path reaches it. |
| T-08-02 | Information Disclosure | DevTools console | mitigate | Keep console clean — no sensitive data exists; verify no error spam on the stress strip. |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | This plan installs NO packages — Kaplay is pre-vendored + pinned (3001.0.19, sha256-verified, Phase 7). No new dependency surface. |

This is a fully local, static, offline single-player browser game: no network, no backend, no auth, no untrusted input beyond keyboard, no storage (localStorage arrives in Phase 11). Attack surface is near-zero; the register reflects that honestly.
</threat_model>

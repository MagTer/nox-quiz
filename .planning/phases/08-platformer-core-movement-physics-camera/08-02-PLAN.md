| collision events → respawn state | `onCollide("checkpoint")` updates closure state from engine-generated events (author-defined entities, not untrusted data). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | Tampering | checkpoint/respawn closure state | accept | State lives in the scene closure, mutated only by engine collision events on author-defined "checkpoint" entities; no external/untrusted input path. |
| T-08-04 | Denial of Service | respawn loop | mitigate | Respawn repositions to a valid checkpoint above the fall threshold and zeroes velocity, so it cannot re-trigger instantly into an infinite respawn loop. |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | No packages installed — Kaplay pre-vendored + pinned (Phase 7). No new dependency surface. |

Fully local, static, offline single-player browser game: no network, no backend, no auth, no storage (Phase 11), no untrusted input beyond keyboard. Surface is near-zero.

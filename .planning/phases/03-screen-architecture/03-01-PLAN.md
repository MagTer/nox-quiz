  </read_first>
  <action>
    Make two additions to math-lab.html:

    ADDITION 1 — CSS (insert immediately before the closing `</style>` tag):

    Add a comment block and the following CSS rules:

      /* -------------------------------------------------------
         Screen visibility system — data-screen / data-panel
         All panels hidden by default; CSS shows the active one.
         No JavaScript display toggling outside renderScreen().
      ------------------------------------------------------- */
      [data-panel] {
        display: none;
      }

      main[data-screen="quiz"]          [data-panel="quiz"]          { display: block; }
      main[data-screen="dungeon-map"]   [data-panel="dungeon-map"]   { display: flex; }
      main[data-screen="combat"]        [data-panel="combat"]        { display: flex; }
      main[data-screen="loot"]          [data-panel="loot"]          { display: flex; }
      main[data-screen="floor-summary"] [data-panel="floor-summary"] { display: flex; }
      main[data-screen="dead"]          [data-panel="dead"]          { display: flex; }

      /* Dungeon placeholder panels — centered layout, full-height */
      [data-panel]:not([data-panel="quiz"]) {
        flex-direction: column;
        align-items: center;
        justify-content: center;
        min-height: 100vh;
        padding: 80px 24px 40px;
        gap: 16px;
      }

      [data-panel] h2 {
        font-size: 2rem;
        font-weight: 900;
        color: var(--accent);
        letter-spacing: 0.06em;
        text-transform: uppercase;
      }

      [data-panel] p {
        font-size: 1rem;
        color: var(--text-muted);
        font-family: Arial, sans-serif;
      }

    The quiz panel uses display:block because the existing #game-board flex layout is defined on main — the quiz section is a block child that fills it. Dungeon panels use display:flex to center their placeholder content independently.

    ADDITION 2 — HTML (insert immediately before `</main>`, after the closing `</section>` of the quiz panel added in Task 1):

      </section>

      </section>

    All text is set via element content (not innerHTML in JS) and is safe static markup.
    Do NOT add any inline style attributes to these sections — styling is handled entirely by the CSS rules added above.
    Do NOT modify anything in the script block.
  </action>

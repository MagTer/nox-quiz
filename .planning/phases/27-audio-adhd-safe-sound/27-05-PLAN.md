<objective>
Wire the gesture-gated music start and the per-scene mute key/icon into all three scenes (title, select, game) — the last piece of code needed before the phase's audio system is fully live end-to-end.

Purpose: AUD-02 requires music to start only after the player's first title-screen press, never before. AUD-03 requires M to toggle mute "anywhere" — since Kaplay 3001's go() clears the entire app-wide input event bus on every scene transition (already engine-verified in this project's Phase 22-03 finding), the mute key and its visual indicator must be re-registered fresh in every scene body, not once at boot.
Output: title.js starts music synchronously inside its existing dual-input start handler and wires the mute UI; select.js and game.js each wire the mute UI (plus a belt-and-braces ensureMusicPlaying() re-assert, a no-op once music is already playing).
</objective>

<task type="auto">
  <name>Task 1: Create src/parallax.js with pure helper functions</name>
  <files>src/parallax.js</files>
  <action>
    Create `src/parallax.js` that exports two functions. All Kaplay globals (`add`, `sprite`, `pos`, `z`, `width`, `destroyAll`) must be referenced only inside function bodies — no top-level engine references.

    - `makeParallaxLayer(name, bounds, ratio, zLayer, y)`:
      - Compute `levelWidth = bounds.right - bounds.left`.
      - Compute `count = Math.ceil((levelWidth + width() * 2) / width()) + 1` to cover the level plus one strip of padding on each side.
      - Create `count` sprites starting at `x = bounds.left - width() + i * width()` at the given `y`, with `z(zLayer)` and a custom `{ ratio }` property.
      - Return the array of instances.
    - `makeParallaxLayers(bounds)`:
      - Return an array of three layer objects `{ name, instances, ratio }` for far/mid/near using the constants in `CONFIG.PARALLAX` and the y anchor `CONFIG.PARALLAX.Y_ANCHOR` minus the strip height so the bottom of each strip sits near `CONFIG.FLOOR_Y`.
      - Use names `"bg-far"`, `"bg-mid"`, `"bg-near"`.
    - `updateParallaxLayers(layers, camX, bounds)`:
      - For each layer, for each instance at index `i`, set `inst.pos.x = bounds.left - width() + i * width() + camX * (1 - inst.ratio)`.
      - This maps camera motion to the parallax ratio while keeping the layer anchored to the level start.

    The module imports only `CONFIG` from `./config.js`. No `wait()`, `loop()`, `tween()`, or `setTimeout`.
  </action>

  </verify>
  <done>
    `src/parallax.js` exists, exports `makeParallaxLayers` and `updateParallaxLayers`, contains no top-level engine references, and parses cleanly.
  </done>

    Add a new module-private function `bestMarginToPoint(point, nodes, spawnPaths, envelope)` near the existing `bfsWithPathMargin` (not exported, matching that function's own module-private convention). `point` is `{x, y}`. For every node `n` in `nodes` where `spawnPaths.has(n.id)` (the already-computed spawn-reachable set — never re-derive this, pass it in), evaluate TWO candidate types and collect all that succeed:

    (1) Same-surface (trivial-walk) candidate: if `point.x >= n.xStart && point.x <= n.xEnd && Math.abs(point.y - n.y) < 4`, the point sits directly on an already-reachable surface — no jump needed. Candidate marginRatio = `spawnPaths.get(n.id)` (the path-so-far cost; zero extra cost for the walk itself). This branch exists because a same-height "just walk there" case produces NO valid jumpReach candidate at all (the flat-Δy quadratic's only positive root is the FULL ~162px flat-jump distance — there is no "zero-jump" candidate — so a point close to spawn on the same floor would otherwise incorrectly report unreachable).

    (2) Cross-height (real hop) candidate — evaluated only when NOT already covered by (1), and skipped entirely when `point.x < n.xStart` (this game's driver/model is rightward-travel-only; a point behind a node is out of scope for this check, matching the alcove's own "off the required forward path" placement convention): compute `dy = point.y - n.y` and call `jumpReach(dy, envelope)` (the existing export, unmodified — do not re-derive the quadratic). Compute the horizontal-reach window this specific (point, node) pair permits — this is the CORRECTED formula, deliberately different from canReach's own overlap-only window:
      - if `point.x <= n.xEnd` (point's x falls within n's own span): window = `[0, point.x - n.xStart]` (takeoff can be anywhere from directly-under-the-point to n's own far-left edge).
      - else (`point.x > n.xEnd`, a real horizontal gap ahead of n): window = `[point.x - n.xEnd, point.x - n.xStart]`.
    For every jumpReach candidate `{reach}` whose value falls within `[windowMin, windowMax]`, compute `marginRatio = reach / theoreticalMaxReach` using the IDENTICAL theoreticalMaxReach derivation `canReach` already inlines at its own lines ~172-179 (the larger-root quadratic solve at this exact `dy`, using `CONFIG.JUMP_FORCE`/`CONFIG.GRAVITY`/`envelope.runSpeed` — copy that exact formula, do not diverge). Keep the LOWEST such marginRatio for this node, then combine with the path-so-far cost: `Math.max(spawnPaths.get(n.id), hopMarginRatio)`.

    Across ALL nodes and BOTH candidate types, track the MINIMUM combined value found. Return `{ marginRatio: <minimum> }` if any candidate was found anywhere, else `null`.

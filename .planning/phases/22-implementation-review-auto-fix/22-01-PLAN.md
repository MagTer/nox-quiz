    Convert every `strip_comments`-fed grep pipeline in scripts/check-gate.sh from the early-exit quiet form to a read-to-EOF count form, per the deferred-items.md suggested fix. There are 14 pipeline call sites: 10 positive quiet-form assertions (assertions 2, 3a, 3b, 4, 5a, and thin-caller checks 9–13), 1 positive extended-regex quiet form (assertion 5b, the destroy/destroyAll alternation), and 3 negative extended-regex quiet forms inside if-conditions (assertions 6, 7, 8).

    Conversion rules:
    - Positive assertions: use the test form where the count of matches must be greater than zero, e.g. `[ "$(strip_comments "$TARGET" | grep -c 'PATTERN')" -gt 0 ] || fail "..."`. The `-E` variants use `grep -Ec`.
    - Negative assertions (6, 7, 8): convert the if-condition to `[ "$(strip_comments "$TARGET" | grep -Ec 'PATTERN')" -gt 0 ]` so a found banned token still triggers `fail`. This also fixes a latent false-GREEN race in the negative assertions: under pipefail, an early-exiting quiet-mode grep that FOUND a banned token could SIGPIPE the upstream sed and make the pipeline report failure, silently skipping the `fail` branch.
    - Every grep PATTERN string stays byte-identical. Every `fail` message stays byte-identical. Assertion order unchanged. `set -euo pipefail` stays. `strip_comments` stays.
    - Update the script's header comment: note the assertions now read stdin to EOF and cite the Phase 21 diagnosis conceptually ("pipefail + early-exit quiet-mode grep SIGPIPE race, diagnosed in Phase 21 deferred-items.md, commit 32348f5"). Do NOT reproduce the old pipeline form verbatim in any comment — the verify step for this task negative-greps the quiet-mode flags, and an explanatory comment echoing them would trip it.

    Commit as an atomic fix commit per 22-PATTERNS.md "Atomic fix commit format": subject `fix(22-01): de-flake check-gate.sh SIGPIPE race`, body = symptom (~30% spurious FAIL on clean tree) → root cause (pipefail + early-exit quiet grep SIGPIPEs upstream sed) → what the fix does (count-form reads to EOF) → what it deliberately does not change (patterns, messages, assertion order, pipefail).

# Summary: Add a Reset Progress control to the title screen

## What changed

**`src/progress.js`** — added `resetSave()`, a new export guarded identically
to the existing `writeSave()`/`loadSave()`: silent no-op when storage is
unavailable, catch+warn (never rethrow) on failure. Its single storage call is
`localStorage.removeItem(CONFIG.SAVE.KEY)` — never a wildcard whole-storage
wipe — so any future, unrelated persisted key (e.g. an eventual audio
mute-toggle setting) can never be touched by this control.

**`src/config.js`** — added `TITLE.RESET_SIZE/RESET_Y/CONFIRM_PANEL_W/
CONFIRM_PANEL_H/CONFIRM_TITLE_SIZE/CONFIRM_BODY_SIZE/CONFIRM_HINT_SIZE/
RESET_FLASH_MS` layout constants (no magic numbers in the scene).

**`src/scenes/title.js`** — added a muted "press R to reset progress" prompt
(grey, small, visually secondary to the bright-green start prompt) plus a
dark-grunge Y/N/ESC confirm overlay:
- Pressing **R** opens the overlay ("Reset ALL progress?" / body / Y-N-ESC
  hint) and **disarms** the existing Enter/Space/click start controllers for
  its duration — the only safe way to prevent Kaplay's non-scoped global
  `onClick()` from also firing "start" on the same click/keypress while the
  overlay is up.
- **N / ESC** tears down only the overlay's own tagged objects, re-arms start,
  touches no data.
- **Y** calls `resetSave()`, tears down the overlay, re-arms start, and shows
  a brief self-destroying "Progress reset." flash (tween + `onEnd(destroy)`,
  the same no-timer idiom `hud.js`'s `flashLevelUp()` already uses).

**`scripts/check-progress.sh`** — two new positive assertions (resetSave
exported from progress.js; wired into title.js) and one negative assertion
(no `localStorage.clear()` anywhere in progress.js).

## Why this approach

Kaplay's global `onClick(cb)` is an unscoped raw mouse-press listener with no
z-order occlusion (confirmed by reading the vendored `lib/kaplay.mjs`), so a
clickable "Reset Progress" button would ALSO fire the pre-existing
click-anywhere "start" handler on the same click. The control is therefore
keyboard-only (R, never a clickable box), and the confirm overlay actively
cancels/re-arms the start controllers around its own lifetime rather than
relying on click occlusion that Kaplay does not provide.

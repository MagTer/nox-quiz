# Quick 260717-j24: Re-play Polish — Enlarge Goal Flag to 32px + Lower Clear SFX Summary

Two play-test polish fixes: the level-end goal flag now renders at 32px (was 16px, "looked like half a coin") while its onCollide trigger collider stays byte-identical at 16px, and the level-complete clear.ogg now plays at 0.4 gain instead of ear-piercing full SFX volume.

## What Was Built

**Task 1 — decoupled goal flag (`src/config.js`, `src/levels/build.js`), commit `8e1727c`:**
- Added `GOAL_VISUAL_SIZE: 32` in config.js immediately next to the unchanged `GOAL_SIZE: 16`, with comments marking GOAL_SIZE as the harness-critical load-bearing collider footprint.
- Replaced the single sprite+area()+"goal" entity in build.js with the door-pattern split:
  - (a) cosmetic `sprite("goal", {width/height: GOAL_VISUAL_SIZE})`, `anchor("botleft")`, `pos(g.goal.x, g.goal.y + GOAL_SIZE)`, tagged `"goalflag"`, NO area() — base planted on the original 16px ground line, grows up and right.
  - (b) hidden `rect(GOAL_SIZE, GOAL_SIZE)` at `pos(g.goal.x, g.goal.y)` with `opacity(HIDDEN)` + `area()` + `"goal"` tag — byte-identical origin and size to the old sprite-derived collider, so browser-boot / audit-endgate-key onCollide is unaffected.
- No scale() used; GOAL_SIZE unchanged; all engine globals stay inside the builder function body (a727c13).

**Task 2 — lower clear SFX (`src/config.js`, `src/ui/mathGate.js`), commit `da1cd94`:**
- Added `CLEAR_VOLUME: 0.4` to CONFIG.AUDIO following the existing `JUMP_VOLUME` per-SFX-override idiom.
- Changed `audio.playSfx("clear")` → `audio.playSfx("clear", CONFIG.AUDIO.CLEAR_VOLUME)` in mathGate.js; `playSfx(name, vol)` already accepts an optional volume, CONFIG already imported — no new imports.
